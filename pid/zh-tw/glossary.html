1受控體：船及其模型

① 直覺 你在操控什麼

「受控體」就是被控制的那個東西。對我們而言它是一艘靠舵轉向的船，而貫穿整門課的核心事實是這個：舵設定的是轉向速率，不是航向。下游的一切都是這件事的後果。

$$ T\,\dot r + r = K\,(\delta + d), \qquad \dot\psi = r $$

這就是 Nomoto 模型：從舵到偏航角速度（yaw rate）的一個一階（first-order）延遲，再從偏航角速度到航向的一個積分。令 $\dot r = 0$，穩態轉向速率就是 $r_\infty = K(\delta + d)$ — 除非淨舵為零，否則船會永遠轉下去。

術語意義
受控體（plant）被控制的系統。這裡指 USV 的轉向動態 — 舵進、航向出。控制器的工作，是讓受控體在自身延遲與擾動之下仍照你的意思動。
USV無人水面載具（unmanned surface vessel）— 我們要維持在某個航向上的自主船。它代表任何靠轉向速率轉彎的東西：四旋翼的偏航、汽車的航向、戰車的砲塔。
Nomoto 模型標準的一階船舶轉向模型，$T\dot r + r = K\delta$。是仍表現得像一艘真實船的最簡單方程式：偏航角速度透過一個時間常數落後於舵。
$K$ — 轉向增益（turning gain）每單位舵對應的穩態偏航角速度，單位 $\frac{\text{rad/s}}{\text{rad}}$。我們這艘船：$K=0.6$，所以 10° 舵會產生約 $\sim 6$ °/s 的穩態轉向。
$T$ — 時間常數（time constant）船體（hull）響應有多遲鈍，單位為秒。我們這艘船：$T=2.5$ s。經過一個 $T$ 後，偏航角速度達到最終值的 $\sim 63\%$；經過 $3T$ 後則達 $\sim 95\%$。
舵角 $\delta$（rudder angle）你實際下達的控制輸入（rad 或度）。不是航向 — 舵命令的是轉向。
偏航角速度 $r$（yaw rate）船轉得有多快，$\dot\psi$，單位 rad/s（或 °/s）。舵透過 $K$ 與 $T$ 直接控制它。
航向 $\psi$（heading）船首（bow）指向哪個方向，單位 rad（或度）。它是偏航角速度的積分，$\psi=\int r\,dt$ — 這正是為什麼固定的舵會讓航向永遠斜坡上升。
擾動 $d$（disturbance）來自外部的推力 — 水流或風 — 建模為作用在舵上的一個固定偏移量，在我們這艘船上是 $\equiv 4°$ 卡住的舵。它的作用恰如一個控制器必須抵消掉的偏置。

💡 撐起其他所有術語的那個事實

舵 → 速率，速率 →（積分後）航向。船會帶著動量衝進你所指的方向，而且它的反應是慢半拍的。延遲＋累積＝超調（overshoot）。這份詞彙表大部分內容，都是用來對抗這一句話的詞彙。

2三個項：P、I、D 以及它們各自讀作什麼

② 數學 控制律及其單位

我們在微控制器上每個 $dt$ 跑一次的離散控制律，誤差為 $e = \psi_\text{set} - \psi$：

$$ u_n = k_p\,e_n + k_i \underbrace{\textstyle\sum_k e_k\,dt}_{\text{積分}} + k_d \underbrace{\frac{e_n - e_{n-1}}{dt}}_{\text{微分}}, \qquad u \leftarrow \mathrm{clamp}(u,\,-\delta_\max,\,+\delta_\max) $$

每個增益都有單位，而從物理意義去讀懂它們，能讓整定不再那麼玄：

術語意義
設定值 $\psi_\text{set}$（setpoint）你想要船維持的航向。控制器追逐的那個參考值。
誤差 $e$（error）$e = \psi_\text{set} - \psi$，單位 rad。進入 PID 前一律先做角度纏繞（見 §3）。零誤差是目標；符號告訴你該往哪邊轉。
$k_p$ — 比例增益（proportional gain）單位 $\frac{\text{rad 舵}}{\text{rad 誤差}}$（無因次）。讀作「我偏差每一度，要打多少舵」。$k_p$ 越大＝響應越快、偏差越小、振鈴越多 — 一直到它再也停不下來為止。
$k_i$ — 積分增益（integral gain）單位 $\frac{\text{rad 舵}}{\text{rad}\cdot\text{s}}$（即 $1/\text{s}$）。「我對持續存在的誤差施壓的力道有多猛」。它應該是迴圈裡最慢的東西 — 它是一份記憶，而 $k_i$ 太大就意味著從短記憶裡就跳結論。
$k_d$ — 微分增益（derivative gain）單位 $\frac{\text{rad 舵}}{\text{rad/s}}$（即秒）。「擺動每秒一度，要打多少舵」。它只對變化起作用；一旦上了航向且穩定下來，$D=0$。
P 項（$k_p e$）彈簧（spring）。力與你偏差多遠成正比。強勁、即時，但在誤差為零時它輸出零舵 — 所以它撐不住擾動。
I 項（$k_i \textstyle\sum e\,dt$）配平片（trim tab）。累積遲遲不消的誤差，並把舵爬升到任何能抵消擾動的穩態值。在我們這艘船上，它會學會水流所需的 $-4°$ 配平，然後在誤差為零時維持住。
D 項（$k_d \,\dot e$）阻尼器（damper）。對抗快速變化，所以它替接近過程剎車，並消滅超調。誤差快速縮小時它為負 — 那個抵消就是阻尼。
控制步驟（每 dt） pseudocode
error      = wrap(setpoint - heading)        # rad，永遠先纏繞（見 §3）
integral  += error * dt                       # rad·s，跨呼叫持續保留
derivative = (error - prev_error) / dt        # rad/s

u = kp*error + ki*integral + kd*derivative    # 舵的 rad 數
u = clamp(u, -RUDDER_MAX, +RUDDER_MAX)
prev_error = error
∑ 在漂亮的接近過程中，P 與 D 互相抵消

操船途中（航向 60°、設定值 90°、$k_p=2$、$k_d=3$）：$P=+1.047$ rad 往裡推，但船首已經以 $\sim 20$ °/s 收斂，所以 $D=-1.047$ rad 往回拉。它們恰好互相抵消，控制器幾乎是在滑行 — 再用力推只會超調得更兇。那個抵消正是 D 存在的全部意義。

3失效模式與其修法

④ 真實案例 每個人都會遇到的 bug

這些是天真的 PID 在真實船上出包的幾種有名字的方式 — 以及每一種的標準修法。它們每一個，都是 §1 那兩個事實與真實致動器、感測器相撞之後的後果。

術語意義
穩態偏差（steady-state offset）純 P 控制器最終停留下來的那截殘餘誤差。要維持航向對抗水流，舵必須維持在 $-4°$，但 $u=k_p e$ 只能靠讓 $e_\text{ss} = -4°/k_p \ne 0$ 來做到這件事。P 是靠維持錯誤來保持航向的。由 I 修正。
超調（overshoot）在穩定下來之前先衝過設定值。成因是帶著動量衝進目標再加上延遲。由 D 馴服（並且別把 P 操過頭）。
整定時間（settle time）船要花多久才會待在設定值附近一個小區間之內。整定時你拿來跟超調與抖動互相權衡的那個量。
積分飽和（積分捲繞，windup）當舵飽和時，I 項持續累積它根本無從施展的誤差。在一次 0°→180° 的轉向裡，舵被釘在 $+35°$ 約 $\sim 9$ s；積分能存下 $\sim 13$ rad $\approx 750°$ 的命令舵角。船於是永遠在繞圈圈。由抗積分飽和修正。
抗積分飽和（anti-windup）任何能在飽和期間阻止積分膨脹的方案。三種標準形式：箝位（clamp）（把積分封頂，讓 $k_i\cdot\text{integral}$ 無法超過舵的極限）、條件積分（conditional integration）（在已飽和的取樣點上不要積分）、反算法（back-calculation）（每一步從積分中減去 $(u_\text{unclamped}-u_\text{clamped})\cdot k$ — 業界標準，最平滑）。
飽和（saturation）致動器（actuator）撞上它的物理極限，$\pm 35°$ 舵。命令可以要求更多；舵就是給不出來。它是積分飽和的根本成因。
微分衝擊（derivative kick）當設定值階躍時，誤差在一個取樣點內就跳一截，所以 $\dot e$ 暴衝。一次 10° 的設定值階躍會給出 $10°/0.05\,\text{s} = 200$ °/s 的假微分 $\approx 600°$ 的舵 — 一個沒用又把伺服機猛甩的尖峰。船並沒有突然開始擺動；是你的設定值移動了。
對量測值微分（derivative-on-measurement）衝擊的修法：對 $\psi$ 微分，而不是對 $e$。當設定值固定時 $\dot e = -\dot\psi$，所以它是同一個阻尼訊號 — 但它會忽略設定值的跳動。翻一個符號就好：d = -(meas - prev_meas)/dt。
低通濾波器（low-pass filter）替微分對抗感測器雜訊做平滑。雜訊的微分非常大（$\sigma=0.5°$ 雜訊 → $\sim 14$ °/s 的垃圾 $\to \times k_d \approx 42°$ 的舵抖動）。濾波器 $d_f \leftarrow \alpha\,d_f + (1-\alpha)\,d_\text{raw}$（$\alpha\approx0.85$）把它砍掉，代價是替 D 加一點延遲。
角度纏繞（angle wrapping）把一個角度誤差映射進 $(-\pi,\pi]$，好讓船走近路轉。航向 350°、設定值 10° 其實只差 20°，但天真的相減卻給出 $-340°$ —「轉遠路」，在碼頭邊就是撞船。wrap(a) = (a+π) % (2π) − π，套用在每一個誤差上。
三種修法，用程式碼寫 Python
import math

def wrap(angle):                                  # → (-pi, pi]，走近路
    return (angle + math.pi) % (2 * math.pi) - math.pi

# 對量測值微分（消除設定值階躍時的微分衝擊）
derivative = -(measurement - prev_measurement) / dt

# 對微分做低通濾波（消除雜訊放大）
d_filtered = alpha * d_filtered + (1 - alpha) * d_raw   # alpha ≈ 0.85

# 抗積分飽和，箝位形式：把積分封頂，讓 ki*integral
# 永遠無法命令出超過舵所能給出的量
integral = max(-RUDDER_MAX / ki, min(RUDDER_MAX / ki, integral))
⚠️ 那兩個在出事之前都看不見的東西

角度纏繞與一份陳舊的積分，都會通過每一個剛好沒觸發到它們的測試 — 然後在船第一次跨過正北、或閒置後重新接管時撞船。每一個誤差都要纏繞；重新接管時要 reset() 控制器。

4整定：增益、極限與配方

⑤ 實作 如何挑增益

整定是一門挑選 $k_p, k_i, k_d$ 的藝術，目標是又快、又平滑、又沒有偏差的響應。底下的詞彙就是配方所用的語言。

術語意義
臨界增益 $K_u$（ultimate gain）讓閉迴路以等幅振盪的那個 $k_p$（純 P，$k_i=k_d=0$）。在我們這艘船上 $K_u \approx 5$。它是 Ziegler–Nichols 的參考基準點。
臨界週期 $T_u$（ultimate period）在 $K_u$ 處那個持續振盪的週期。在我們這艘船上 $T_u \approx 9$ s。它餵給 ZN 公式。
Ziegler–Nichols一條經典整定法則，從 $K_u$ 與 $T_u$ 算出增益：$k_p=0.6K_u$、$k_i=1.2K_u/T_u$、$k_d=0.075K_uT_u$。由我們的 $K_u=5,\,T_u=9$ 算出 $k_p=3.0,\,k_i=0.67,\,k_d=3.4$。它以快速的擾動抑制（disturbance rejection）為目標，並容忍超調 — 是一個火力很猛、可以從這裡往保守方向退調（de-tune）的起點。
相位延遲（phase lag）命令與效果之間的延遲：伺服機變化率、感測器濾波延遲、通訊延遲時間（latency）、船體動態。高增益下的相位延遲，就是引發振盪的萬用配方。延遲越多 → 可用的 $k_p$ 越低。它正是為什麼「把 $k_p$ 拉到它開始振盪」這招行得通的原因 — 即使單看理想的 Nomoto 受控體永遠不會振盪。
變化率限制（slew rate）致動器能移動的最快速度，我們的伺服機是 $25$ °/s — 所以從一頭打到另一頭要 $\sim 2.8$ s。在高增益下，命令在兩端之間翻動的速度比舵跟得上的還快，把一個方波命令變成落在後面的鋸齒波：這就是相位延遲的具體現身。
🛠️ 配方，照順序來

（1）$k_i=0,\,k_d=0$；把 $k_p$ 拉到出現持續振盪（$\approx K_u=5$），然後砍半到 $k_p \approx 2\text{–}2.5$。（2）把 $k_d$ 拉到超調消失（$\approx 3$）。（3）加上能在合理時間內消除偏差的最小 $k_i$（$0.2$，約在 $\sim 30$ s 內把水流配平掉）。結果：$k_p=2,\ k_i=0.2,\ k_d=3$。實地的順序一模一樣：P → D → I，從低開始。

5部署：從模擬到一艘真正的 USV

④ 真實案例 在水上

上硬體時數學不變；變的是管路接法。這些是當控制器離開模擬器、在一個 ROS 2 節點裡跑起來時，你會遇到的術語。

在模擬器裡在船上（ROS 2）
psi（航向）來自 IMU／/odometry/filtered 的偏航 — 在上游已由 EKF 或互補濾波器（complementary filter）融合好了。
wrap(sp - psi)一樣的纏繞，在你的節點裡每一週期、對每一個誤差都做。
DT = 0.05計時器回呼（timer callback）約 $\sim 20\text{–}50$ Hz — 但要用量測到的 $dt$，不是標稱值。
舵 $\delta$一具舵伺服機，或差動推力（differential thrust）：left = base + u、right = base - u。
RUDDER_MAX你真實的致動器極限 — 在軟體裡也要箝位，並搭配抗積分飽和。
水流擾動 $d$真實的水流與風 — 配平掉它，是 I 項的常設工作。
🔑 任何自主航行測試前的兩條不可妥協的鐵律

要有一個硬體緊急斷電開關（kill switch），且已接好、測試過。而且每次控制器重新接管時都要 reset() PID — 一份五分鐘前留下的陳舊積分，就是一個正等著把船甩向碼頭的積分飽和 bug。

6快速檢查

Q1 把症狀對到藥方
一艘調成 PD 的船在穩定水流中維持在 92°，而不是命令的 90°，而且永遠補不上最後那 2°。哪一個詞彙表術語點出了問題，又是哪一個能修好它？
  • 微分衝擊；由對量測值微分修正。
  • 穩態偏差；由加上積分（I）項修正。
  • 積分飽和；由箝位積分修正。
  • 角度纏繞；由纏繞誤差修正。
一個 P（或 PD）控制器輸出 $k_p e$，所以要維持抵消水流所需的 $-4°$ 舵，它就必須保持一個非零的誤差：$e_\text{ss} = -4°/k_p$。那就是穩態偏差。D 只對變化起作用，在穩態時歸零，所以幫不上忙。積分會把舵爬升到配平值，並讓誤差回到零 — 這正是 I 存在的目的。（積分飽和與衝擊是 I 側與設定值階躍的問題；纏繞是走近路的問題 — 都不符合這個症狀。）
