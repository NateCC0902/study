I 控制：配平片

你的 PD 控制器把航向（heading）守得漂漂亮亮 — 卻在自欺欺人。它停在離設定值（setpoint）差個幾度的地方就喊收工，因為那段偏差（offset）是 P 唯一能讓舵（rudder）抵著水流（current）撐住的辦法。本章談的就是那個能修好它的項：積分（integral），也就是控制器裡那個會去學會擾動（disturbance）、再把它配平（trim）掉，好讓誤差（error）終於能歸零的部分。

⏱ 約 30 分鐘🧭 案例 4⚡ 即時模擬∑ 積分項

1遺留下來的問題：PD 是靠維持錯誤來保持航向的

① 直覺 上回我們停在哪裡

我們已經做出了一個不錯的 PD 控制器：$k_p=2$ 帶我們到位，$k_d=3$ 讓我們不超調（overshoot）地滑進去。它就是第 03 章裡那個泊車小弟控制器。但它有個缺陷，只在出現持續性擾動時才會現形 — 一道水流、一股穩定的側風（wind）、一個配平失衡。回想一下那艘船：模型說水流的作用恰恰像一個卡住 4° 的舵在推著船首（bow）。

💡 配平片（trim tab）的類比

在真實的船或滑翔機上有一片小小的配平片 — 一個你只設定一次的次要小面板，好讓主控制面能停在中立位置，同時仍抵著一股穩定的力撐住。積分項就是自動駕駛儀（autopilot）替自己調整的那片配平片。它默默地捲入恰好足夠的偏置去抵消水流，讓比例（proportional）項不必再去跟它對抗。

要維持固定航向，淨偏航（yaw）力矩必須為零。水流是一個固定的等效舵角（rudder angle）$+4°$，所以控制器裡某個東西必須維持一個固定的 $-4°$ 來抵消它。一個純 P 控制器只能輸出 $k_p e$ — 而它要產生 $-4°$ 的舵角，唯一的辦法就是停在一個非零的誤差上：

$$ k_p\,e_{ss} = -4° \quad\Longrightarrow\quad e_{ss} = \frac{-4°}{k_p} $$

在 $k_p=2$ 時那就是 $e_{ss}=-2°$：你叫它去 $90°$，船卻穩定在 $92°$。D 幫不上忙 — 一旦你停住不動，微分（derivative）就是零，所以 D 毫無貢獻。PD 是靠維持錯誤來保持航向的。這不是整定（tuning）失敗；這是結構性的。誤差為零時 PD 控制器輸出零舵，而零舵贏不了水流。

⚠️ 你不能只是猛拉 $k_p$

那個誘人的修法 — 把 $k_p$ 拉大來縮小偏差 — 會一頭撞上第 01–02 章的那堵牆：這艘船在 $k_p\approx5$ 附近就會變得不穩定（即臨界增益（ultimate gain, Ku）$K_u$）。用可用的增益時，這個偏差是永久性的。我們需要的是一個根本上不同的機制，而不是一根更硬的彈簧（spring）。

2積分到底是「為了什麼」

① 直覺 控制器長出了一份記憶

比例項對你現在身在何處作出反應。微分項對你正往哪裡去作出反應。兩者對過去都毫無記憶。積分項就是控制器的記憶：它隨時間累積誤差。如果一個小誤差賴著不走，積分就會持續增長，越壓越用力，直到它累積出足夠的輸出把誤差推到零 — 然後它就停在那裡，即使誤差已經消失，仍把那個偏置撐住。

flowchart LR E["誤差 e = psi_set - psi"] --> P["P: kp e（現在）"] E --> D["D: kd de/dt（趨勢）"] E -->|"隨時間累加 e*dt"| I["I: ki 積分（歷史）"] P --> U["舵 u"] D --> U I --> U U --> B["船"] C["水流 d = +4 度"] --> B B -->|"航向 psi"| E
② 數學 為什麼這能逼出零誤差

底下這個論證說明積分讓穩態誤差（steady-state error）恰好為零，而不只是縮小。假設閉迴路（closed loop）穩定下來 — 每個訊號都歸於定值。積分項是 $i = k_i \int_0^t e\,d\tau$。要讓 $i$ 為定值，它的變化率必須為零：

$$ \frac{di}{dt} = k_i\, e(t) = 0 \quad\Longrightarrow\quad e(t)=0 $$

這就是整個訣竅。積分要停止增長，唯一的辦法就是誤差為零。所以只要還有任何誤差殘留，積分就會持續移動舵；它只有在把誤差驅趕到一無所有時才能歸於靜止。而且關鍵在於，它歸於靜止時並不會鬆開 — 它會把累積出來的那個偏置撐住。那個被撐住的值就是抵消水流的配平。

∑ 配平最後落在哪裡

在新的穩態下船回到了 $90°$，所以 $e=0$，因此 $P=0$、$D=0$。水流仍以 $+4°$ 推著，所以舵仍必須維持 $-4°$ — 而唯一還剩下能供應它的項就是 I。積分已經斜坡升到恰好 $k_i\,\text{積分} = -4°$ 並停泊在那裡。積分已經把水流學會了。

3看它把水流學會

④ 真實案例 案例 4，即時

這就是筆記裡的案例 4，真的跑起來。船一開始就已經穩定在 $90°$。在 $t=20$ s 時水流開啟 — 一個階躍擾動，等於那個卡住 $4°$ 的舵。把航向面板和舵面板放在一起看。

🧭 PID — 擾動在 t=20 s 開啟，I 把它配平掉
🔑 拖什麼、看什麼

設定 $k_p=2$、$k_d=3$，並從 $k_i=0$ 開始（純 PD）。在 $t=20$ s 時水流踢進來，船被推歪到約 $92°$ 然後就停在那裡 — 那就是 §1 裡的結構性偏差。現在把 $k_i$ 拉到 $0.2$ 再跑一次：踢進來之後，航向下探到大約 $91.7°$，然後在約 $30$ s 內慢慢爬回恰好 $90.0°$。在舵面板裡，你真的能看到那個穩定的舵偏置一路滑到 $-4°$ — 那就是積分在把配平填進去，同時比例的貢獻放鬆回到零。

一旦 $k_i=0.2$，有兩個數字可以直接從圖上讀出來：

  • 恢復過程是故意慢的 — 大約 $30$ s。積分本來就應該對那些持續存在的東西作出反應，而不是對每一道掠過的浪。慢在這裡才是對的。
  • PD 那條軌跡永遠不會恢復 — 它就一直停在 $92°$。同一艘船、同一個擾動；唯一的差別就是有沒有一個記憶項在扛著配平。
⚠️ 現在把 $k_i$ 也拉得太過頭

把 $k_i$ 猛拉到 $2$ 再跑一次。船在恢復時超調，然後在 $90°$ 兩側緩慢來回蛇行好一陣子才安定下來。那就是積分從一段短記憶裡就跳結論 — 它累積偏置的速度比遲緩的船體（hull）能跟上的還快，於是超調，接著只好再鬆開。這預告了 §4：$k_i$ 必須是迴圈裡最慢的東西。

4為什麼 $k_i$ 必須小 — 迴圈裡最慢的東西

① 直覺 記憶 vs. 反射

把這三個項想成三種不同時間尺度的決策。P 是反射 — 對當下的瞬間反應。D 是預判 — 對眼前趨勢的反應。I 是判斷 — 從累積的證據裡得出的結論。判斷應該是最後才改變主意的那個，而不是最先。一個記憶項抽動得跟反射一樣快的控制器，就是一個會對雜訊（noise）和短暫的暫態驚慌失措的控制器。

🔑 積分應該是迴圈裡最慢的東西

$k_i$ 大 = 從短記憶裡就跳結論 = 超調與緩慢蛇行。$k_i$ 小 = 有耐心，只去壓那些真正持續存在的誤差。積分的全部價值就在於它忽略暫態、只學那個穩定的偏置。把它弄快，你就毀掉了你當初加它進來所要的那唯一一個性質。

② 數學 積分會增添延遲

這條經驗法則背後有個硬道理，不只是品味問題。積分會引入相位延遲（phase lag）：它的輸出落在誤差之後，因為它是過去的累積。在頻率視角下，一個積分器貢獻 $-90°$ 的相位。相位延遲正是吃掉你穩定裕度的東西 — 它跟讓高 $k_p$ 振盪（oscillation）的是同一個機制（伺服機（servo）的延遲，第 02 章）。所以積分在本質上就是一個會破壞穩定性的項。你買到了零穩態誤差，代價則付在相位裕度上。讓 $k_i$ 保持小，帳單就保持小。

$$ G_I(j\omega) = \frac{k_i}{j\omega} \quad\Rightarrow\quad \angle G_I = -90°,\qquad |G_I| \propto \frac{1}{\omega} $$

注意那個幅值：$|G_I|\propto 1/\omega$。積分器在低頻有巨大的增益（這就是它能輾碎那個固定偏差的原因），而在高頻有很小的增益（這就是它會明智地忽略快速碎浪的原因）。那份低頻的權威是它的特色；$-90°$ 的相位是它的代價。整定的配方 — 「加上能在合理時間內消除偏差的最小 $k_i$」— 不過就是在這兩者之間取得平衡。

$k_i$$t=20$ s 踢進來之後的行為結論
0永久 $2°$ 偏差，永遠不恢復這就是那個 bug
0.2約 $30$ s 內恢復到 $90.0°$，不蛇行選定的增益
2超調，蛇行數十秒太熱 — 記憶太愛跳結論

5程式裡的積分

③ 程式碼 每一行都讀

積分項只有兩行，但兩行都很要緊。那個累加器跨呼叫持續保留 — 那正是它之所以是記憶而不是公式的原因。而且增益 $k_i$ 乘的是整個累積總和，不是當下的誤差。

控制步（每個 dt） Python
error      = wrap(setpoint - heading)        # rad，永遠先纏繞（第 07 章）
integral  += error * dt                       # rad*s，跨呼叫持續保留
derivative = (error - prev_error) / dt        # rad/s

p = kp * error
i = ki * integral                             # <- 配平片：學會偏置
d = kd * derivative

u = p + i + d                                  # 舵的 rad 數
u = clamp(u, -RUDDER_MAX, +RUDDER_MAX)
prev_error = error

追蹤一下這三項貢獻在案例 4 實驗中的變化。$t=20$ s 之前，船在 $90°$ 上，誤差為零：P=0、I=0（什麼都還沒累積）、D=0，舵歇在 $0°$。水流開啟；船首飄走，誤差增長，P 反推回去並在 $\approx92°$ 把它攔住。現在 integral += error*dt 開始每一步往上爬，因為誤差非零、而且持續非零。隨著積分增長，$i = k_i\,\text{積分}$ 接手扛起這份負荷，舵偏置滑向 $-4°$，誤差縮回零，P 淡出 — 而積分一發現 $e=0$，就恰好停在原地不再增長。交接完成。

pid_controller.py Python
class PID:
    def __init__(self, kp, ki, kd, output_limits=(-1.0, 1.0)):
        self.kp, self.ki, self.kd = kp, ki, kd
        self.out_min, self.out_max = output_limits
        self.integral = 0.0
        self.prev_error = None

    def reset(self):
        """重新接管時呼叫：陳舊的積分就是一個積分飽和 bug（第 05 章）。"""
        self.integral = 0.0
        self.prev_error = None

    def update(self, error, dt):
        p = self.kp * error
        self.integral += error * dt                 # 記憶在這裡增長
        i = self.ki * self.integral
        d = 0.0 if self.prev_error is None else self.kd * (error - self.prev_error) / dt
        self.prev_error = error
        u = p + i + d
        return max(self.out_min, min(self.out_max, u))   # 箝位
🛠️ 箝位藏著一個陷阱

注意，即使輸出已被箝位（clamp）在舵止（rudder stop），積分仍持續累積。對於溫和的案例 4 擾動，這無傷大雅 — 舵根本不會飽和（saturation）。但對於一次大幅度操船，這正是積分飽和（積分捲繞，windup）（第 05 章）的誕生方式：記憶不斷往一堵它看不見的牆灌滿。下一章我們會正面迎戰那個 bug。眼下，就先好好品味積分在誠實地做它的本分吧。

6手算一個配平片時間步

② 數學 都是數字，不打馬虎眼

取水流開啟後幾秒鐘、積分還在學習途中的一個快照。增益 $k_p=2,\ k_i=0.2,\ k_d=3$。船被推歪到了 $91.5°$（設定值 $90°$），它幾乎靜止不動，而積分到目前為止已累積了 $-0.20$ rad·s 的負誤差。一個時間步：

此刻 P 仍扛著大部分的工作（$-3.0°$），而 I 才只累積到 $-2.3°$。但每一步積分都變得更負一點，同時隨著誤差收斂，P 縮小。快轉到穩態：誤差 $\to0$，所以 P $\to0$、D $\to0$，而積分已經斜坡升到 $k_i\,\text{積分} = -4°$，全靠它自己。整個過程裡舵的總量始終維持在 $-4°$ 附近 — 改變的只是組成，從由 P 供應轉為由 I 供應。那個轉移，就是配平片把負荷從你手裡接過去。

7換你上場

Q1 觀念檢查
為什麼加上積分項能把穩態航向誤差驅趕到恰好為零，而不只是像更大的 $k_p$ 那樣單純把它縮小？
  • 因為 $k_i$ 比 $k_p$ 大，所以它主導了輸出。
  • 在穩態下，積分唯有在誤差為零時才會停止變化，所以迴圈只能在 $e=0$ 時歸於靜止。
  • 因為一旦船停下來，積分就抵消了微分項。
  • 因為水流擾動過一陣子之後會自己消退。
一個定值的積分要求 $\tfrac{di}{dt}=k_i e=0$，這就逼出 $e=0$。被撐住的積分值接著供應那 $-4°$ 的配平來抵消水流，而 P 與 D 都歸零。更大的 $k_p$ 只能把偏差除以 $k_p$（$e_{ss}=-4°/k_p$）— 它永遠到不了零，而且還沒接近零，你就先用光了可用的 $k_p$。
Q2 算一算
用一個純 PD 控制器（$k_i=0$）、$k_p=2$，配上一道等於卡住 $4°$ 之舵的水流，穩態航向偏差的大小是多少，以度為單位？（提示：$k_p e_{ss}=-4°$。）
°
$e_{ss} = -4°/k_p = -4°/2 = -2°$，所以船穩定在偏離 $2°$ 之處 — 被命令去 $90°$ 卻停在 $92°$。要從一個 P 項裡擠出 $-4°$ 的舵，你就必須停在一個 $-2°$ 的誤差上。加上 $k_i$ 就能消除它，因為最後扛著配平的是積分，而不是誤差。
Q3 推理 — 為什麼「慢」才是特色
在 $k_i=0.2$ 的模擬裡，$t=20$ s 踢進來之後的恢復大約要 $30$ s。把 $k_i$ 拉到 $2$ 會讓它超調並蛇行。請以積分所代表的意義為出發點，解釋為什麼慢的那個版本才是正確的設計 — 以及是什麼物理原因使得 $k_i=2$ 行為失常。
提示
積分是累積誤差的一份記憶。它應該對哪幾種誤差作出反應，又該忽略哪幾種？另外回想一下，一個積分器會替迴圈增添 $-90°$ 的相位延遲。
顯示一種解答

積分存在的目的是抵消那些持續存在的擾動 — 一道穩定的水流、一個配平失衡 — 而不是像單一道浪那樣的暫態。一個慢的積分會隨時間平均，只去壓那些維持非零的誤差，而那正是水流的樣子。那份耐心，就是我們加它進來所要的那個性質。

把 $k_i$ 猛拉到 $2$，有兩件事會壞掉。第一，記憶現在會對它本該忽略的短暫態作出反應，於是它在遲緩的船體（時間常數（time constant）$T=2.5$ s）能反應過來之前就先累積出偏置 — 它把配平超調了，接著只好再鬆開，產生那緩慢的蛇行。第二，積分器貢獻 $-90°$ 的相位延遲；把 $k_i$ 灌大，你就花掉了穩定裕度，把迴圈推向振盪。解方就是那個配方：加上能在可接受時間內消除偏差的最小 $k_i$，然後就停手。積分應該是迴圈裡最慢的東西。

🔑 本章回顧

一個 PD 控制器是靠維持錯誤來保持航向的：要對著水流推出 $-4°$ 的舵，P 就必須停在 $e_{ss}=-4°/k_p$，一個你用可用的增益整定不掉的結構性偏差。積分修好了它 — 它成為控制器的記憶，累積誤差，而且唯有在 $e=0$ 時才能停止增長，所以它把穩態誤差驅趕到恰好為零，並停泊在水流所要求的那 $-4°$ 配平上。在模擬裡，擾動在 $t=20$ s 階躍進來；PD 被推歪了就一直歪著，PID 則下探後在約 $30$ s 內恢復。那份遲緩就是特色，也正是為什麼 $k_i$ 必須是迴圈裡最小、最慢的那一項 — 你每加一分積分，都買到了低頻權威，卻也花掉了相位裕度。下一章：當那份不斷增長的記憶撞上一個飽和的舵時會發生什麼事 — 積分飽和，積分的黑暗面。
