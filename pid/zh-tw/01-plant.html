1這艘船：同一個模型，貫穿每個案例

① 直覺 先把畫面建起來

我們的「受控體（plant）」——控制術語裡指被控制的對象——是一艘維持羅盤航向（heading）的無人水面載具（USV）。你不能直接命令航向；你命令的是舵角（rudder angle） $\delta$。舵讓船體（hull）以某個偏航角速度（yaw rate）$r$ 轉向，而航向 $\psi$ 就是這份轉向累積出來的結果。這一句話正是後面所有困難的根源。

💡 一句話的模型

舵設定的是轉向速率，不是航向。航向是轉向的累計總和——所以船總是帶著動量衝進你所指的方向。

我們採用最簡單、卻仍像真船的模型：一階（first-order）Nomoto 模型。它說偏航角速度透過單一時間常數（time constant）落後於舵，而航向是偏航角速度的積分。

flowchart LR D["舵 δ"] -->|"增益 K、延遲 T"| R["偏航角速度 r"] R -->|"∫ dt"| P["航向 ψ"] W["水流／風"] --> R
② 數學 推導它

Nomoto 方程與航向積分式，其實就是兩條互相耦合的一階常微分方程：

$$ T\,\dot r + r = K\,(\delta + d), \qquad \dot\psi = r $$

其中 $K$ 是穩態轉向增益（每單位舵角對應的偏航角速度），$T$ 是時間常數（船體響應有多遲鈍），$d$ 是一個固定擾動（disturbance）——水流（current）或風（wind）——它的作用就跟一個卡在偏移位置的舵一模一樣。讀一下穩態：令 $\dot r = 0$，這艘船就會以 $r_\infty = K(\delta+d)$ 永遠轉下去。不存在任何能讓 $\dot\psi = 0$ 的航向，除非 $r = 0$，也就是除非舵加擾動的總和為零。

2開迴路：船自己會怎麼動

① 直覺 放開方向盤

把舵固定在一個角度，雙手放開，看著它。這裡沒有控制器——只有受控體。拖動滑桿，讀這兩格面板：上方是航向，下方是舵。

🧭 開迴路航向（固定舵、無控制器）
🔑 該注意的地方

偏航角速度不會瞬間跳到它的最終值——它是爬上去的，在 $t=T=2.5$ s 時達到約 63%（這正是一階延遲（lag）的行為）。而航向永遠不會穩定：它會一直斜坡上升，因為航向是一個非零轉向速率的積分。把舵設成 0° 它才終於停下——但這樣你也哪兒都去不了了。

④ 實際案例 為什麼重要

所有靠速率轉向的載具都是這樣：船、四旋翼的偏航（yaw）、汽車航向、戰車砲塔。你永遠沒辦法在開迴路下「設定航向」；你得把舵的時機抓得分毫不差，再分毫不差地把它收回來。只要一有擾動，你就偏了。這正是我們要閉合迴路（close the loop）的理由。

3導致一切的兩個事實

把這兩點記在腦子裡——後面每一章都是在跟它們纏鬥：

  1. 舵控制的是速率，不是航向。你命令的是轉向；航向是累積出來的。所以船會帶著動量衝進目標。
  2. 響應（response）有延遲。不管舵現在做了什麼，船體還是會把舊的動作再持續個幾秒（$T=2.5$ s）。
⚠️ 延遲＋累積＝超調

帶著動量衝進設定值，加上有延遲的響應，就是超調（overshoot）與振鈴（ringing）的萬用配方。好的控制器不會抹去這兩個事實——它會預判它們。

4你將要實作的控制律

② 數學 離散 PID

連續 PID 是 $u(t) = k_p e + k_i\!\int e\,dt + k_d \dfrac{de}{dt}$，誤差（error）$e = \psi_\text{set} - \psi$。我們在微控制器上每隔 $dt$ 跑一次，所以要離散化：積分變成累加和，微分（derivative）變成有限差分。

$$ u_n = k_p e_n + k_i \underbrace{\textstyle\sum_k e_k\,dt}_{\text{積分}} + k_d \underbrace{\frac{e_n - e_{n-1}}{dt}}_{\text{微分}}, \qquad u \leftarrow \mathrm{clamp}(u,\,-\delta_\max,\,+\delta_\max) $$
控制步驟（每個 dt） 虛擬碼
error      = wrap(setpoint - heading)        # rad，永遠先纏繞（第 07 章）
integral  += error * dt                       # rad·s，跨呼叫持續保留
derivative = (error - prev_error) / dt        # rad/s

u = kp*error + ki*integral + kd*derivative    # 舵的 rad 數
u = clamp(u, -RUDDER_MAX, +RUDDER_MAX)
prev_error = error

這三個增益都有單位，從物理上去讀它們，能讓整定少很多神祕感：

增益單位讀作
$k_p$rad 舵 / rad 誤差「我偏差每一度，要打多少舵」
$k_i$rad 舵 / (rad·s)「我對持續存在的誤差施壓的力道有多大」
$k_d$rad 舵 / (rad/s)「擺動每秒一度，要打多少舵」

6用程式碼寫出 PID

③ 程式碼 每一行都讀

這是控制器寫成一個小而可重用的類別——跟你會放進 ROS 2 節點（node）的形狀一模一樣。其中的狀態（integral、prev_error）正是讓它成為一個控制器、而不只是一條公式的關鍵：它會記憶。

pid_controller.py Python
class PID:
    def __init__(self, kp, ki, kd, output_limits=(-1.0, 1.0)):
        self.kp, self.ki, self.kd = kp, ki, kd
        self.out_min, self.out_max = output_limits
        self.integral = 0.0
        self.prev_error = None          # None = 尚無前一次取樣

    def reset(self):
        """控制器在關閉後重新接管時呼叫。"""
        self.integral = 0.0
        self.prev_error = None

    def update(self, error, dt):
        """一個控制步驟。error 已經做過角度纏繞（rad）。"""
        p = self.kp * error
        self.integral += error * dt
        i = self.ki * self.integral
        d = 0.0 if self.prev_error is None else self.kd * (error - self.prev_error) / dt
        self.prev_error = error
        u = p + i + d
        return max(self.out_min, min(self.out_max, u))   # 箝位
🛠️ update() 裡頭藏著兩個設計選擇

7換你了

Q1 觀念檢查
在開迴路模擬裡，航向為什麼會一直斜坡上升、而不是穩定在某個角度？
  • 這個模型不穩定——任何舵都會讓它發散爆掉。
  • 航向是一個非零偏航角速度的積分，所以它會無上界地累積。
  • 擾動會隨時間不斷變大。
  • 舵的變化率限制讓它停不下來。
固定的舵給出固定的穩態偏航角速度 $r_\infty = K\delta$。於是航向 $\psi = \int r\,dt$ 永遠線性爬升。只有 $r = 0$（淨舵為零）才能讓它停下——這正是為什麼你需要回授（feedback）才能維持一個航向。
Q2 算一算
在 $K = 0.6$ (rad/s)/rad、舵固定為 $10° = 0.1745$ rad 時，穩態偏航角速度是多少度每秒？（提示：$r_\infty = K\delta$，再換算單位。）
°/s
$r_\infty = 0.6 \times 0.1745 = 0.1047$ rad/s $= 6.0$ °/s。船體在 $T = 2.5$ s 後達到約 63%（3.8 °/s），$3T$ 後約 95%。
Q3 推導——把積分項離散化
連續的 I 項是 $k_i\int_0^t e(\tau)\,d\tau$。請說明它如何在程式裡變成累加和 integral += error*dt，並指出哪一個變數必須在每次呼叫之間持續保留。
提示
在 $[0,t]$ 上的定積分，就是 $e(\tau)$ 之下的面積。用寬 $dt$、高 $e_k$ 的矩形去近似那塊面積。
看一個解答

矩形（左黎曼和）近似：$\int_0^{t_n} e\,d\tau \approx \sum_{k=0}^{n} e_k\,dt$。每一步都在前一個總和上加一個矩形 $e_n\,dt$，這正是 integral += error*dt。那個累加器 self.integral 必須跨 update() 呼叫持續保留——它就是控制器對「誤差已經滯留多久」的記憶。每次控制器重新接管時就把它重置（reset()），否則幾分鐘前留下的陳舊累加和會變成一個積分飽和（windup）bug。

🔑 本章回顧

這艘船是一個一階 Nomoto 受控體：舵 →（有延遲的）偏航角速度 →（積分而成的）航向，再加上一個固定水流。開迴路下它會永遠斜坡上升——你必須閉合迴路。離散 PID 控制律把 P、I、D 加總起來並箝位到舵的極限；它的狀態（integral、prev_error）就是它的記憶。下一章：只開 P 項，認識那根彈簧。
