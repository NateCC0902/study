D 控制：阻尼器

單靠 P 會超調並振鈴 — 它是一根沒有摩擦的彈簧（spring）。在這一章裡，我們把 $k_p=2$ 固定下來，再加上微分（derivative）項。我們會看 D 在收斂途中如何對抗 P，精準釘出控制器決定滑行的那一刻，並證明 D 永遠做不到的那一件事。

⏱ ~30 分鐘🧭 PD 航向保持⚡ 即時模擬🛑 阻尼

1為什麼 P 需要一個搭檔

① 直覺 我們上次停在哪

上一章只有 P 的船表現得就像彈簧上的一顆質量（mass）。下令從 0° 轉到 90°，你離設定值（setpoint）越遠，舵（rudder）就拉得越用力 — 完全就像彈簧拉得越開、回拉的力就越大。麻煩在於一根輕阻尼的彈簧會做的事：它會振盪（oscillate）。船首（bow）擺向 90°，但它抵達時速度很快，直接衝過去，然後振鈴（ring）。

具體來說，用我們敲定的那組增益（gain），$k_p=2$ 振鈴一次後穩定在約 92°。推到 $k_p=5$ 它就永遠停不下來 — 那是這艘船的臨界增益（ultimate gain, Ku）$K_u$，一個 $\pm16°$ 的振盪（oscillation）永無止境。解法不是更多彈簧，而是摩擦。那就是微分項。

💡 一句話講完 D 的模型

P 對你身在何處（誤差（error））起反應。D 對那個位置變化得多快（誤差的變化率）起反應。一個避震器（shock absorber）不在乎車子在哪裡 — 只在乎懸吊移動得多快。D 就是那個避震器。

② 數學 我們正在加入的那一項

我們沿用第 01 章那個離散控制律（control law），只是把 $k_d$ 打開。微分是（已纏繞過的）誤差的有限差分：

$$ u_n = \underbrace{k_p\,e_n}_{\text{P：位置}} \;+\; \underbrace{k_d\,\frac{e_n - e_{n-1}}{dt}}_{\text{D：速度}}, \qquad e_n = \psi_\text{set} - \psi_n $$

仔細看符號。當船正擺向設定值時，誤差正在縮小，所以 $e_n < e_{n-1}$，有限差分為負。在 $k_d > 0$ 的情況下，D 因此會是負的 — 它把舵往跟 P 相反的方向推。D 在收斂過程中對抗 P。這個對抗就是整個機制的全部。

2親手操作：自己把阻尼器加上去

④ 實例 在你信任數學之前，先感受它

一如既往的同一艘船：$K=0.6$、$T=2.5$ s、20 Hz 迴圈、$\pm35°$ 舵。P 增益釘死在 $k_p=2$。這裡唯一重要的旋鈕是 $k_d$。跑一個 0° → 90° 的階躍，盯著航向（heading）面板看。

🧭 PD 航向保持（自己加上阻尼器 kd）
🔑 拖什麼、看什麼

掃描（sweep）$k_d$，重現筆記裡的三種情況：

  • $k_d=0$ — 純 P。它超調（overshoot）到大約 118° 並振鈴。這是你最醜的基準線。
  • $k_d=1$ — 一個小而有禮的超調。阻尼器（damper）開始咬住了。
  • $k_d=3$ — 它像泊車小弟一樣滑進去。完全沒有超調。

現在當船首接近 90° 時，盯著舵面板看：在 $k_d=3$ 時，命令舵角實際上會變成負的 — 反舵（counter-rudder）— 而且早在船抵達之前就發生了。那個負的尖峰就是 D 在煞車。P 絕不會在還沒到達目標時就下令反舵；只有 D 會。

🛠️ 把它推過頭

把 $k_d$ 拉向 8–10。收斂過程變得遲鈍，舵也開始顯得神經兮兮。太多阻尼器會讓船害怕自己的運動 — 而且（預告第 07 章）它會把感測器雜訊（noise）放大成舵的抖動。有一個甜蜜點，對這個船體（hull）來說 $k_d=3$ 就是它。

3D 對抗 P：重讀那個實作範例

② 數學 用數字看抵消

D 對抗 P 最乾淨的證明，就活在第 01 章那個手算的時間步（timestep）裡。回想那個操船途中的快照 — 航向 60°、設定值 90°、前一次誤差 31°、增益 $k_p=2,\ k_d=3$：

步驟計算結果

P$2 \times 0.5236$+1.047 rad（+60°）
$\dot e$$(0.5236 - 0.5411)/0.05$−0.349 rad/s（船首正以 20 °/s 收斂）
D$3 \times (-0.349)$−1.047 rad（−60°）
總和（P + D）$1.047 - 1.047$≈ 0 rad → 控制器滑行
∑ 再讀一次這個結果

船離設定值還有30°，然而 P（+60° 的舵）與 D（−60°）卻恰好抵消。控制器命令出基本上為零的舵。為什麼？因為船首已經以 20 °/s 收進來。再用力推也不會更快到 — 只會累積出更多動量，超調得更兇。那個抵消就是阻尼（damping）。對照只有 P 的世界：單靠 P 會要求 +60°，被箝位（clamp）到 +35° 的舵止，全速衝進牆裡。

這就是為什麼模擬器（simulator）裡的 PD 船早在抵達之前就鬆手。D 正在讀取收斂速率，並悄悄地把它從需求中減掉。你收斂得越快，它減掉的就越多。

4滑行法則：它何時歸於中性？

② 數學 推導那個門檻

上面那個抵消不是運氣。控制器歸於中性 — P + D = 0 — 恰好發生在兩項大小相等、方向相反的時候。把它們對立起來：

$$ k_p\,e = k_d\,\dot e \quad\Longrightarrow\quad \dot e = \frac{k_p}{k_d}\,e $$

（我在這裡把 $\dot e$ 寫成一個正的收斂速率；在收斂途中 $\dot e$ 其實是負的，平衡的是兩者的大小。）代入我們的增益 $k_p=2$ 與 $k_d=3$：

$$ \dot e = \frac{k_p}{k_d}\,e = \frac{2}{3}\,e $$
🔑 經驗法則

在 $k_p=2,\ k_d=3$ 時，一旦收斂速率（°/s）超過剩餘誤差（°）的 ⅔，控制器就停止推力。所以在離目標 30° 時，門檻是 $\tfrac{2}{3}\times 30 = 20$ °/s — 收斂快過這個值，D 就贏，舵鬆手。那正是那個實作範例：離 30°、以 20 °/s 收斂，恰好站在 P 與 D 抵消的刀鋒上。

比值 $k_p/k_d$ 就是控制器心目中「每剩一度該有的安全收斂速度」。把它調低（更多 $k_d$，或更少 $k_p$），船就堅持要慢慢地爬進去 — 重阻尼。把它調高（更少 $k_d$），它就願意快速抵達 — 欠阻尼，會超調。整定（tuning）D，就是在選擇要讓船多麼積極地滑進它的目標。

5D 只對變化起作用 — 以及這付出的代價

① 直覺 D 的盲點

這是最重要的單一限制，而且它直接從數學裡掉出來。D 與 $\dot e$（誤差的變化率）成正比。船一旦穩定到某個航向並維持住，誤差就停止變化：$\dot e = 0$，於是 D = 0。運動一停止，阻尼器就在那一瞬間消失。那正是你想要阻尼器做的事 — 但這也意味著 D 對於維持一個穩定的航向毫無貢獻。

⚠️ D 修不了偏差，也完成不了轉向

回想案例 2：水流（current）推船首的力道像 4° 卡住的舵，而一個 P 控制器會帶著一個固定的偏差（offset）穩定下來，因為在誤差為零時它命令出零舵。加上 D 對這一切毫無改變。當船坐在它那個（錯誤的）穩定航向上時，誤差是固定的，$\dot e=0$，於是 D = 0 — 它無從抓取。D 只馴服運動；它無法供應一個穩定擾動（disturbance）所要求的那個固定舵角。消滅那個偏差完全是另一項的工作（積分（integral）— 下一章）。

② 數學 兩個邊界條件

D 的兩種失敗都是同一個事實看了兩次。在一個穩定的偏差航向上，誤差是某個固定值 $e_\text{ss} \neq 0$：

$$ \dot e = 0 \;\Rightarrow\; \text{D} = k_d\,\dot e = 0, \qquad u_\text{ss} = k_p\,e_\text{ss} + 0 = k_p\,e_\text{ss} $$

穩態的舵角單由 P 決定 — D 什麼都沒加。所以面對水流時，PD 落在跟只有 P 一樣的偏差上。D 讓旅程平順；它不改變終點。把這句話記著：它是 I 項的懸念。

6程式碼裡的 D 項

③ 程式碼 每一行都讀

這是第 01 章那個一模一樣的 PID 類別 — 我們只是緊盯著那兩行本身就是微分的程式。注意對最開頭那個取樣點（sample）的防護：還沒有 prev_error 時，有限差分是未定義的，所以 D 被強制設為零。跳過那個防護，你的第一個時間步就會拿 None 去算出一個垃圾微分，然後把舵猛甩到底。

pid_controller.py Python
def update(self, error, dt):
    """One PD step. error is already angle-wrapped (rad)."""
    p = self.kp * error

    # --- 微分項 ----------------------------------------------
    # 第一個取樣點：沒有前一次誤差，所以變化率未定義。
    # 強制 D = 0，而非拿 None 去做差分。
    if self.prev_error is None:
        d = 0.0
    else:
        de = error - self.prev_error          # 這一步的誤差變化量
        d  = self.kd * de / dt                # rad 舵；收斂過程中為負
    self.prev_error = error                    # 記下來給下次呼叫用
    # ---------------------------------------------------------

    u = p + d                                  # （ki*integral 在第 04 章加入）
    return max(self.out_min, min(self.out_max, u))   # 箝位到舵止
④ 實例 驗證那個實作範例

把同一組數字餵進去做個健全性檢查 — 這應該要精準重現手算的結果。注意我們在每個誤差抵達控制器之前就先纏繞（wrap）它（那個角度纏繞（angle wrapping）的 bug 自成一章），而且我們把真實的迴圈週期當作 dt 傳進去。

check_worked_example.py Python
import math

pid = PID(kp=2.0, ki=0.0, kd=3.0, output_limits=(-0.611, 0.611))  # ±35°
DT  = 0.05

# 用「前一次」取樣播種控制器：航向 59°、設定值 90°
prev_heading = math.radians(59.0)
pid.update(math.radians(90.0) - prev_heading, DT)   # prev_error = 31° = 0.5411 rad

# 現在是快照那一步：航向 60°、設定值 90°
e = math.radians(90.0) - math.radians(60.0)         # 30° = 0.5236 rad
u = pid.update(e, DT)

print(f"P  = {pid.kp * e:+.3f} rad   ({math.degrees(pid.kp*e):+.1f}°)")
print(f"de = {e - math.radians(31.0):+.4f} rad   over dt = {DT}s")
print(f"u  = {u:+.3f} rad   ({math.degrees(u):+.1f}°)")
# P  = +1.047 rad   (+60.0°)
# de = -0.0175 rad  -> D = 3 * (-0.0175/0.05) = -1.047 rad (-60.0°)
# u  = +0.000 rad   (+0.0°)   <- P 與 D 抵消：船滑行
⚓ 在真正的 USV 上

在水上，$\dot e$ 來自於對一個 20 Hz、帶雜訊的羅盤做差分 — 而雜訊的微分大得驚人（相鄰兩次 0.5° 的讀數就能算出 14 °/s 的幻影擺動）。這就是為什麼生產級程式會對量測值（measurement）微分，而不是對誤差微分，並對它做低通濾波（low-pass filter）。這兩件事都是你在這裡的 update() 裡做的決定；我們會在第 06–07 章遇到逼出它們的那些 bug。現在，理想感測器，乾淨的 D。

7換你了

⑤ 練習 把這些法則變成你自己的
Q1 觀念檢查
PD 船（$k_p=2,\ k_d=3$）頂著水流維持一個穩定的航向，卻穩定在偏離設定值 2° 的地方 — 跟只有 P 的船一樣的偏差。為什麼加上 D 並沒有消除那個偏差？
  • $k_d=3$ 太小了；更大的 $k_d$ 就能補上這個差距。
  • 在穩定航向上誤差是固定的，所以 $\dot e = 0$ 且 D = 0 — D 在維持航向時毫無貢獻，留下 P 獨自對抗水流。
  • D 加上了它自己的偏差，剛好互相抵消。
  • 船穩定下來的同時水流變大了。
D 與 $\dot e$ 成正比。船一旦穩定，$\dot e = 0$，於是 $u_\text{ss} = k_p e_\text{ss} + k_d\cdot 0 = k_p e_\text{ss}$ — 與只有 P 完全相同。D 讓收斂過程平順，卻無法供應穩定擾動所要求的那個固定舵角。那是積分的工作（第 04 章）。再多的 $k_d$ 也改變不了終點。

°/s
$\dot e = \tfrac{2}{3}\times 20° = 13.33$ °/s。在此之下，P 仍主導，舵持續推進轉向；在此之上，D 主導，控制器開始煞車。注意這個門檻會隨著你離得多遠而縮放 — 離 30° 時是 20 °/s，離 20° 時是 13.3 °/s。你越接近，控制器對你的速度就越謹慎。
Q3 推理 — 那個負的舵尖峰
在 $k_d=3$ 的模擬裡，當船還沒到 90° 並正朝它轉去時，舵命令會短暫地變成負的（反舵）。解釋為什麼，並解釋為什麼一個純 P 的控制器永遠做不到這件事。
提示
在收斂的尾端把 $u = k_p e + k_d \dot e$ 寫出來。誤差 $e$ 仍然是正的（你還沒抵達）。$\dot e$ 是什麼符號？而當船首快速擺進來時，D 相對於 P 有多大？
顯示一個解答

在收斂的尾端，誤差又小又是正的（$e > 0$，仍未到達），而收斂速率很大，所以 $\dot e$ 是強烈的負值。於是 $\text{D} = k_d\dot e$ 是一個很大的負數，而 $\text{P} = k_p e$ 是一個小的正數。它們的總和 $u = k_p e + k_d\dot e$ 變成負的：控制器在船抵達設定值之前就下令反舵來煞住這個擺動，扼殺掉那本來會造成超調的動量。

一個純 P 的控制器算的是 $u = k_p e$。只要 $e > 0$（還沒到目標），$u$ 永遠是正的 — 它無法下令反舵，直到船已經超調、$e$ 變號為止。到那時動量已經累積起來，你就振鈴了。預先煞車是 D 獨有的貢獻：它對變化率起作用，所以它能在位置誤差告訴它之前就先反應。

🔑 本章回顧

D 是阻尼器：$\text{D}=k_d\,\dot e$。在收斂途中誤差正在縮小，所以 $\dot e < 0$，D 對抗 P，在船抵達之前就煞住擺動 — 那就是實作範例裡的 P+D 抵消（離 30°、以 20 °/s 收斂，P 與 D 都是 ±60°，淨值為零）。滑行法則 $k_p e = k_d\dot e$ 給出中性門檻：在 $k_p=2,\,k_d=3$ 時，一旦收斂快過剩餘誤差的 ⅔ 就鬆手。但 D 只對變化起作用：一旦穩定，$\dot e=0$ 且 D=0，所以它修不了頂著水流的固定偏差，也完成不了轉向 — 它只馴服運動。在誤差為零時還要撐住舵，需要的是積分。下一章：I，配平片（trim tab）。
