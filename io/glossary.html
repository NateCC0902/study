1Core I/O 核心 I/O

The bedrock: the boundary between your program and the kernel, and the abstraction — the file descriptor — that everything else hangs off. If these are fuzzy, nothing above makes sense.

Term中文One-line definition
System call (syscall)系统调用The controlled doorway from user space into the kernel; the CPU traps, switches privilege, runs kernel code, returns. read, write, ioctl are syscalls.
User space / kernel space用户态 / 内核态Two privilege rings. Your process runs unprivileged; only the kernel touches hardware. The syscall is the only legal crossing.
File descriptor (fd)文件描述符A small non-negative integer indexing your process's open-file table. 0/1/2 are stdin/stdout/stderr; everything you open gets the next free one.
VFS (Virtual File System)虚拟文件系统The kernel layer that makes files, sockets, devices and pipes all answer the same read/write/close API — the machinery behind "everything is a file."
inode索引节点The on-disk (or in-memory) record of a filesystem object's metadata — type, permissions, and for a device node, the major/minor pair. Not the name; names live in directories.
Blocking I/O阻塞 I/OA read() that puts your thread to sleep until data arrives. Simple, but one slow device stalls the whole thread.
Non-blocking I/O非阻塞 I/OOpen with O_NONBLOCK; a read() with no data returns immediately with -1 and errno == EAGAIN instead of sleeping. The prerequisite for an event loop.
Partial read / short read部分读取read() may return fewer bytes than asked — it is not an error. Robust code loops until it has a full message or hits EOF.
Buffering (stdio)缓冲The C library hoards bytes in a userspace buffer and flushes in big chunks, so one fwrite ≠ one syscall. Great for throughput, deadly for latency if you forget to flush.
Raw / unbuffered I/O无缓冲 I/OCalling read/write directly — one syscall per call, no library hoarding. What you want for a 200 Hz sensor where staleness is a bug.
Page cache页缓存The kernel's RAM cache of file/block data. A write() to a file lands here first and is flushed to media later — fast, but the bytes aren't durable until fsync().
errno错误码The thread-local integer a failing syscall sets to say why. EINTR, EAGAIN, ENODEV are the ones you'll handle on a robot.
EINTR系统调用被中断"Interrupted system call" — a signal arrived mid-syscall. Not a real failure; retry the call.
🔑 The amortization formula you keep coming back to

Every syscall costs a fixed overhead (trap + privilege switch + return), call it $C_s \approx 1\text{–}3\,\mu\text{s}$. If you read $B$ bytes per call at a per-byte cost $c_b$, the effective rate is

$$ R_\text{eff} = \frac{B}{C_s + B\,c_b} \xrightarrow{B \to \infty} \frac{1}{c_b} $$

Small reads spend all their time in $C_s$; big reads amortize it away. This single fact is why stdio buffers, why you batch sensor reads, and why io_uring exists.

2Devices & drivers 设备与驱动

How a lump of silicon becomes a file you can open(). The driver is the translator; /dev is the phone book; /sys is the live map of what's plugged in.

Term中文One-line definition
/dev设备目录The directory of device nodes. Each node is a name carrying a (type, major, minor) triple that routes open() to a driver. Populated at boot by devtmpfs.
sysfs (/sys)系统文件系统An in-RAM filesystem exposing the kernel's device model as browsable files. cat an attribute to read a register-ish value; echo to one to poke hardware.
procfs (/proc)进程文件系统Sibling of sysfs for process and kernel state — /proc/interrupts, /proc/cpuinfo, /proc/PID/fd.
Character device字符设备A byte-stream device — UART, I²C bus, GPIO, camera. No page cache between you and the driver. Shows as c in ls -l.
Block device块设备A fixed-block, random-access, cached device — eMMC, NVMe, SD card. Goes through the block layer and page cache. Shows as b.
Major / minor number主设备号 / 次设备号The two integers in a device node. Major selects the driver; minor selects which instance. Packed into a dev_t.
dev_t设备号The 32-bit (modern: 12-bit major + 20-bit minor) packed device number. Encode with MKDEV(maj, min); decode with MAJOR()/MINOR().
devtmpfs设备临时文件系统The tmpfs the kernel auto-mounts on /dev at boot and pre-populates with nodes as drivers register, so /dev exists before udev runs.
udev / udevd设备管理器The userspace daemon that listens for kernel uevents and applies rules — renaming, symlinking, setting permissions. How you get a stable /dev/gps.
uevent设备事件A netlink message the kernel emits on add/remove/change of a device. udev's input.
Kernel module (.ko)内核模块Driver code loaded into the running kernel at runtime with modprobe/insmod — no reboot. Inspect with modinfo, list with lsmod.
In-tree / out-of-tree主线 / 树外驱动In-tree drivers ship with the kernel source and rebuild with it; out-of-tree drivers (vendor blobs, DKMS) are compiled separately and break on kernel upgrade.
Firmware固件A binary blob the driver loads into the device (Wi-Fi, GPU) at probe time via request_firmware(), from /lib/firmware. A missing file shows as error -2 (ENOENT) in dmesg.
Device tree (DT)设备树A data structure (.dts → .dtb) that describes non-discoverable hardware to the kernel — addresses, IRQs, compatible strings — so the right driver binds. Essential on ARM SoCs like the Orin.
ioctl设备控制The catch-all syscall for device commands that don't fit read/write — set a serial baud, query a CAN bitrate, fire a custom command. ioctl(fd, REQUEST, &arg).
🏭 Real case — the USV's Wi-Fi is the whole second column

On the AGX Orin, the onboard Wi-Fi module sits on SDIO (a non-discoverable bus), so it must be declared in the device tree; its out-of-tree vendor kernel module loads a firmware blob from /lib/firmware; udev then gives the resulting interface a predictable name. Every term in that sentence is a row above — and if any one step fails, dmesg tells you which.

3Buses & enumeration 总线与枚举

The wires hardware actually talks over. The single most useful distinction: buses that find their own devices (PCIe, USB) versus buses where you must tell the kernel what's there (I²C, SPI, SDIO) via the device tree.

Term中文One-line definition
Enumeration枚举The process of discovering what's on a bus and assigning addresses. PCIe and USB enumerate automatically; I²C/SPI/SDIO devices must be declared in the device tree.
PCIePCIe 高速总线High-speed, switched, point-to-point serial bus for NVMe, NICs, GPUs. Self-enumerating; inspect topology with lspci -t.
BDF (Bus:Device.Function)PCIe 地址A PCIe device's address, e.g. 0000:05:00.0 = domain:bus:device.function. The handle lspci prints.
BAR (Base Address Register)基址寄存器A PCIe config-space register telling the OS where to map the device's MMIO window into physical memory.
USB通用串行总线Hot-pluggable, tree-topology, self-enumerating bus. Devices describe themselves via descriptors; see the tree with lsusb -t.
USB descriptorUSB 描述符The self-describing data (idVendor, idProduct, serial, endpoints) a USB device reports on plug-in. udev rules match on these.
I²CI²C 总线Two-wire (SDA/SCL) multi-drop bus for low-rate sensors. 7-bit addressing → 128 addresses (a few reserved). Must be declared in the DT.
SPISPI 总线Four-wire (MOSI/MISO/SCLK/CS) full-duplex bus; faster than I²C, addressed by chip-select line, not by address. Declared in the DT.
SDIOSDIO 总线SD-card electrical bus repurposed for I/O peripherals (notably Wi-Fi). Non-discoverable — must be declared in the device tree.
MMIO内存映射 I/OMemory-Mapped I/O: device registers appear as physical memory addresses, so a CPU load/store reads/writes a register. Access from userspace via mmap and volatile.
compatible string兼容性标识The DT property (e.g. "ti,tmp102") that names a device's type so the kernel can pick the matching driver.
Device tree overlay设备树覆盖A patch (.dtbo) applied on top of the base DT to enable a header pin or attach a sensor without recompiling the whole tree — e.g. via jetson-io.
∑ Why "is this bus fast enough?" is just a comparison

Pick a bus by comparing your required throughput against its ceiling. I²C standard-mode tops out near $100\,\text{kbit/s}$ (fast-mode $400\,\text{k}$); a single PCIe Gen4 lane carries about $1.97\,\text{GB/s}$ usable after 128b/130b encoding. A 16-channel lidar at $1\,\text{Mpoint/s} \times 16\,\text{B/point}$ needs

$$ R_\text{req} = 1{\times}10^6 \cdot 16 = 16\,\text{MB/s} $$

— laughably impossible over I²C, trivial over PCIe or Gigabit Ethernet. Choosing a bus is this one inequality, $R_\text{req} \le R_\text{bus}$, done per device.

4Data movement & multiplexing 数据搬运与多路复用

Two questions: how bytes physically get from the device into RAM (interrupts vs. DMA), and how one thread watches many devices at once without busy-waiting (epoll, io_uring).

Term中文One-line definition
PIO (Programmed I/O)程序控制 I/OThe CPU copies every byte itself, register by register. Simple, but it burns a core during transfers.
Interrupt中断A hardware signal that yanks the CPU off its current work to run an interrupt service routine — "data ready, come deal with it" instead of polling.
IRQ (Interrupt Request line)中断请求The numbered line a device raises an interrupt on. Counts live in /proc/interrupts.
ISR / top half / bottom half中断服务程序 / 上半部 / 下半部The handler. The top half runs fast with interrupts off; heavy work is deferred to a bottom half (softirq/tasklet/threaded IRQ) that can sleep.
DMA直接内存访问Direct Memory Access: a dedicated engine moves bytes between device and RAM without the CPU, which only gets one interrupt when the whole block is done. The key to keeping a control loop responsive under heavy I/O.
Coherent vs. streaming DMA一致性 / 流式 DMACoherent buffers stay cache-consistent automatically (small, long-lived); streaming buffers are mapped per-transfer with explicit cache sync (large, one-shot).
IRQ affinity中断亲和性Pinning an IRQ to specific CPUs via /proc/irq/N/smp_affinity, so interrupt storms don't land on the core running your real-time loop.
Coalescing中断合并Batching many events into one interrupt (e.g. NIC rx-coalesce) to trade a little latency for far less CPU overhead under high rates.
select / poll多路复用 (select/poll)Old syscalls to wait on many fds at once. Cost scales $O(n)$ in the number of watched fds — they rescan the whole set every call.
epollepoll 事件机制Linux's scalable readiness API: register fds once, then each wait costs $O(\text{ready})$, not $O(n)$. Supports level- and edge-triggered modes. The backbone of a sensor manager.
Level vs. edge triggered水平 / 边沿触发Level-triggered keeps reporting an fd while data remains; edge-triggered reports only on the transition to readable, so you must drain it fully (loop until EAGAIN).
io_uringio_uring 异步 I/OModern shared-memory ring (submission + completion queues) for true async I/O with near-zero syscalls per op — the high end of multiplexing.
C10k problemC10k 问题The classic challenge of serving 10 000 concurrent connections; the reason thread-per-fd lost to event loops.
⚠️ The edge-trigger footgun

With epoll in edge-triggered mode, if you read once per event you'll leave bytes in the buffer and never get woken again — the fd looks dead. The contract is: on every wakeup, read() in a loop until you get EAGAIN. Half the "my sensor randomly hangs" bugs are this.

5Networking 网络

From the NIC to the socket fd. The two ideas that bite robots most: the routing table (and who steals the default route) and the bandwidth-delay product (how much data is "in flight").

Term中文One-line definition
Socket套接字A file descriptor that speaks the network. socket() returns an fd you then read/write like any other — the network as a file.
TCP vs. UDPTCP / UDPTCP is reliable, ordered, stream — at the cost of retransmits and head-of-line blocking. UDP is fire-and-forget datagrams — perfect for sensor data where a fresh packet beats a re-sent stale one.
NIC网卡Network Interface Controller — the hardware. The kernel driver hands received frames up the stack via interrupts/DMA and NAPI polling.
Routing table路由表The rules deciding which interface a packet leaves by, matched longest-prefix-first. Inspect with ip route.
Default route默认路由The catch-all entry (0.0.0.0/0) for any destination no specific route matches. There should be exactly one you trust.
Default-route theft默认路由抢占When a second interface (a freshly-connected Wi-Fi) installs its own default route and silently steals all egress traffic — the classic "SSH died after I plugged in Ethernet" bug. Fix with route metrics.
Metric路由度量值A priority number on a route; lower wins. How you make Wi-Fi the preferred default and Ethernet the sensor-only path.
MTU最大传输单元Maximum Transmission Unit — the largest payload a link carries in one frame (Ethernet default $1500\,\text{B}$). Exceed it and packets fragment or drop.
BDP (Bandwidth-Delay Product)带宽时延积Bandwidth × round-trip time = the bytes "in flight" on a link; the minimum socket buffer needed to keep a pipe full.
SO_RCVBUF / SO_SNDBUF套接字缓冲区Socket options sizing the kernel receive/send buffers. Too small caps throughput on a high-BDP link; too big invites bufferbloat.
Bufferbloat缓冲膨胀Oversized buffers hoarding packets, adding latency under load even though throughput looks fine — murder for a control link.
Backpressure背压The signal (full buffer, EAGAIN, TCP window closing) telling a fast producer to slow down so a slow consumer can keep up.
∑ BDP, and why the buffer matters

The bytes a link can hold mid-flight — and thus the socket buffer needed to saturate it — is

$$ \text{BDP} = \text{Bandwidth} \times \text{RTT} $$

A $1\,\text{Gbit/s}$ shore-to-USV link with a $20\,\text{ms}$ RTT holds $1{\times}10^9/8 \cdot 0.020 = 2.5\,\text{MB}$ in flight. If SO_RCVBUF is the default $\sim 200\,\text{kB}$, the sender stalls waiting for ACKs and you'll never see line rate.

6CAN & real-time CAN 与实时

The actuator bus, and what it takes to make a control loop hit its deadline every time. Real-time is about the worst case, never the average.

Term中文One-line definition
CANCAN 总线Controller Area Network — a robust, differential, multi-drop bus for actuators and instruments. Frames carry an ID and up to 8 data bytes (64 in CAN-FD).
Arbitration仲裁How CAN resolves simultaneous transmitters: the message with the numerically lowest ID wins (dominant bit overrides recessive); the loser backs off and retries. Lowest ID = highest priority.
Bit stuffing位填充Inserting an opposite bit after 5 identical bits so receivers stay clock-synced — it makes worst-case frame length a few bits longer than nominal.
SocketCANSocketCANLinux's framework treating a CAN interface as a network socket (AF_CAN), so you read/write CAN frames like any fd and use candump/cansend.
Sample point采样点The fraction of a bit time (typically 75–87.5%) at which the controller samples the level. Must match across all nodes or you get errors.
Termination (60 Ω test)终端电阻A 120 Ω resistor at each end of the bus; with both fitted, measuring across CAN-H/CAN-L on a powered-off bus reads ~60 Ω. Wrong termination = reflections = errors.
Error counters / bus-off错误计数 / 总线关闭Each node counts TX/RX errors; cross 255 and it goes bus-off, silently dropping off the bus until reset. The "actuators stopped responding" symptom.
restart-ms自动重启间隔The SocketCAN option (ip link set can0 type can restart-ms 100) that auto-recovers a bus-off interface after the given delay.
CAN-FDCAN-FDFlexible Data-rate CAN: up to 64-byte payloads and a faster data phase, for higher throughput while keeping CAN's arbitration.
NMEA 2000NMEA 2000The marine instrument protocol layered on CAN at $250\,\text{kbit/s}$ — GPS, depth, wind on a USV.
Real-time实时A guarantee about worst-case response, not speed. A 20 Hz loop is real-time if it never misses its $50\,\text{ms}$ deadline — average latency is irrelevant.
Deadline截止期限The latest time a task may finish to be correct. Missing it in a hard-real-time system is a failure, not just slow.
Jitter抖动The variation in how long a periodic task actually takes from cycle to cycle. Low average + high jitter still blows deadlines.
PREEMPT_RT实时内核补丁The kernel patchset making nearly all kernel code preemptible, slashing worst-case scheduling latency from milliseconds to microseconds.
SCHED_FIFO / rtprio实时调度 / 实时优先级A real-time scheduling policy that runs a thread ahead of all normal tasks until it yields — how the control loop gets the CPU on time.
mlockall锁定内存Locks a process's pages in RAM so a page fault never stalls the real-time loop mid-cycle.
Watchdog看门狗A timer that resets (or fails-safe) the system if the software doesn't "kick" it in time — the last line of defense when a loop hangs.
🏭 Real case — the USV's worst-case budget

The Orin runs the steering PID at 20 Hz, so the deadline is $50\,\text{ms}$. Average loop time is $4\,\text{ms}$ — meaningless. What matters is the worst case: a Wi-Fi IRQ storm landing on the loop's core, a page fault, a logging fsync blocking. Fix the budget with IRQ affinity (move the NIC off the loop's CPU), SCHED_FIFO, mlockall, and a watchdog that fails the rudder to neutral if the loop ever does miss.

GQuick self-test 小测验

If these four feel obvious, the glossary did its job. If not, the linked layer is where to look.

Q1 CAN arbitration
Two CAN nodes start transmitting at the same instant — one with ID 0x0A0, the other with ID 0x320. Which message wins the bus, and what happens to the other?
  • 0x0A0 wins (lower ID = higher priority); the 0x320 sender detects it lost, backs off, and retries automatically.
  • 0x320 wins because the higher number has priority; 0x0A0 is dropped.
  • Both collide and corrupt; each node must wait a random backoff like Ethernet.
  • Whichever node started transmitting first wins regardless of ID.
CAN arbitration is non-destructive and priority-based: a dominant (0) bit overrides a recessive (1), so the numerically lowest ID wins bit-by-bit without any data loss. The loser sees the mismatch, immediately stops, and re-arbitrates on the next bus-idle. See §6 — Arbitration.
Q2 Buses & the device tree
Which of these buses requires you to declare the attached device in the device tree because it cannot discover devices on its own?
  • PCIe — it self-enumerates and you read addresses from lspci.
  • USB — devices report descriptors on plug-in and the kernel matches drivers automatically.
  • SDIO (and I²C/SPI) — non-discoverable, so the hardware must be described in the DT for a driver to bind.
  • None — the kernel always probes every bus and finds everything by itself.
PCIe and USB are enumerable: the bus protocol lets the host walk it and read each device's identity. I²C, SPI, and SDIO have no such mechanism — there's no way to ask "who's at address 0x48?" reliably — so the kernel relies on the device tree to know what's wired up and where. This is exactly why the Orin's SDIO Wi-Fi must be in the DT. See §3 — Enumeration.
Q3 epoll triggering
You use epoll in edge-triggered mode and call read() exactly once per wakeup. Your sensor "randomly stops" delivering data even though it's still sending. Why?
  • Edge-triggered mode is buggy and should never be used for sensors.
  • Edge-triggered fires only on the transition to readable; leftover bytes don't re-fire, so you must read() in a loop until EAGAIN to drain the fd.
  • The fd was never registered with epoll; you need to re-add it after every read.
  • read() in edge-triggered mode returns 0 bytes, so no data ever arrives.
Edge-triggered epoll reports the edge — the moment an fd became readable — not the ongoing level. If you leave bytes unread, there's no new edge, so no new wakeup, and the fd looks dead while data piles up. The rule: on each event, loop read() until it returns EAGAIN. See §4 — Level vs. edge triggered.
Q4 What "real-time" means
Your 20 Hz ($50\,\text{ms}$ deadline) control loop averages $4\,\text{ms}$ per cycle but occasionally spikes to $60\,\text{ms}$. Is this loop real-time?
  • Yes — the average is well under the deadline, so it's comfortably real-time.
  • No — real-time is a guarantee about the worst case; a single $60\,\text{ms}$ spike misses the $50\,\text{ms}$ deadline and is a failure.
  • Yes, as long as the spikes happen less than 1% of the time.
  • It's impossible to tell without knowing the CPU clock speed.
Real-time correctness is defined by the worst-case latency, not the mean. A loop that meets its deadline 999 times and misses once is not hard-real-time. The spike is the bug to chase — usually an IRQ, page fault, or blocking fsync — and the tools to kill it are PREEMPT_RT, SCHED_FIFO, mlockall, and IRQ affinity. See §6 — Real-time.
🔑 Chapter recap

Six layers, one vocabulary: the syscall/fd/VFS boundary turns hardware into files; drivers and the device tree bind silicon to /dev, with /sys as the live map; buses split into self-enumerating (PCIe/USB) versus declared (I²C/SPI/SDIO); DMA and epoll move and multiplex bytes without burning the CPU; routing and BDP govern the network; and CAN plus PREEMPT_RT deliver actuators on a hard deadline. Every kernel log line you'll ever read on the USV is one of these words — now you can decode it.
