感測器與致動器：與真實世界的介接

到目前為止，你的接腳只和 LED 與其他接腳對話。現在它們要面對混亂的物理世界——距離、溫度、市電供電的風扇、轉動的馬達。這個世界會反擊：雜訊、電壓突波、反電動勢（back-EMF）。本章談的是如何可靠地連接它，讓你的機器人能撐過第一次碰撞，讓你的繼電器板能撐過第一次斷電。

1訊號鏈：感測器負責量測，致動器負責動作

① 直覺 先建立整體圖像

每一個會接觸真實世界的嵌入式系統都是一條訊號鏈：物理世界被感測、轉成電壓、經過調理、被 MCU 讀取、做出決策，然後再推送回去給一個致動器，由它改變世界。感測器是這條鏈的輸入端；致動器是輸出端。你的韌體則是夾在中間那一層薄薄的判斷力。

💡 一句話模型

感測器把一個物理量轉換成 MCU 能讀取的電氣訊號；致動器則把 MCU 發出的電氣訊號轉換回物理效應。本章談的一切不是這兩者之一，就是把它們接線到不會互相摧毀的那份紀律。

有兩個觀念主宰了一個連接到底能不能運作。第一個是電壓電平：ATmega328P 接腳在 0–5 V 之間運作良好，但超過約 5.5 V 就會損毀；像 ESP32 這類 3.3 V 板子超過約 3.6 V 就會掛掉。第二個是阻抗——大致來說，就是一個節點能夠提供（source）或汲取（sink）多少電流。感測器的輸出可能是一個微弱、高阻抗的訊號，一條又長又有雜訊的導線就能讓它失真；而 MCU 接腳只能提供約 20–40 mA，遠遠不足以驅動一顆馬達。整門手藝就在於匹配電平與阻抗，讓每一級都真的能驅動下一級。

🔑 接腳不是電源供應器

ATmega328P 的 I/O 接腳額定為約 20 mA 連續電流（絕對最大值 40 mA），整顆晶片總和大約 200 mA。驅動一顆 LED 沒問題。但繼電器線圈（約 70 mA）、伺服機，或直流馬達（數百 mA 到數安培）就不行——這些需要在接腳與負載之間放一顆驅動電晶體或專用晶片。直接用接腳驅動馬達，你不是讓晶片掉電重置（brown out），就是把接腳燒掉。

2量測距離：HC-SR04 與音速

① 直覺 2 美元模組裡的回聲定位

HC-SR04 是一個聲納。你對它的 TRIG 接腳送一個短脈衝；它就發出一段 40 kHz 的超音波「啁啾（chirp）」，然後把 ECHO 接腳拉到 HIGH 並維持高電位，直到啁啾撞到某物反彈回來。那段 ECHO 脈衝的寬度就是聲音的來回飛行時間。把它計時，你就知道障礙物有多遠——就跟蝙蝠一樣。

sequenceDiagram participant MCU participant HCSR04 as HC-SR04 participant Obj as 障礙物 MCU->>HCSR04: TRIG 拉 HIGH 維持 10 µs HCSR04->>Obj: 8 × 40 kHz 脈衝串 HCSR04->>MCU: ECHO 變為 HIGH Obj-->>HCSR04: 聲音反射回來 HCSR04->>MCU: ECHO 變為 LOW (經過 t_echo 之後) Note over MCU: 脈衝寬度 = 來回時間
② 數學 推導距離公式

聲音在 20 °C 乾燥空氣中傳播速度約為 $v \approx 343\ \text{m/s}$。ECHO 脈衝量到的是啁啾飛出去到障礙物再飛回來的時間——也就是距離的兩倍。所以單程距離是飛行路徑的一半：

$$ d = \frac{v \cdot t_{\text{echo}}}{2} $$

現在把它變得實用。Arduino 量到的 $t_{\text{echo}}$ 單位是微秒，而我們想要的 $d$ 單位是公分。把音速換算成 cm/µs：

$$ v = 343\ \tfrac{\text{m}}{\text{s}} = 343\times\frac{100\ \text{cm}}{10^{6}\ \mu\text{s}} = 0.0343\ \tfrac{\text{cm}}{\mu\text{s}} $$

代回距離公式並把常數整理出來：

$$ d_{\text{cm}} = \frac{0.0343 \cdot t_{\mu s}}{2} = \frac{t_{\mu s}}{58.3} \;\approx\; \frac{t_{\mu s}}{58} $$

這就是每一篇機器人教學都會引用的著名「除以 58」規則。它其實就是 $2/0.0343 = 58.3$ 四捨五入後的結果。用 58 取代 58.3 會把距離高估約 0.5%——相較於 $v$ 本身的溫度相依性（每 °C 漂移約 0.17%），這是可以忽略的。如果你需要精準度，請計算 $v = 331.4 + 0.6\,T_{°C}$ m/s，再除以對應的常數。

∑ 範例演算

ECHO 維持 HIGH 達 $t_{\text{echo}} = 1740\ \mu\text{s}$。則 $d = 1740 / 58 = 30\ \text{cm}$。用 SI 單位做合理性檢查：$d = 0.0343 \times 1740 / 2 = 29.9\ \text{cm}$。障礙物大約在 30 cm 外。

③ 程式碼 逐行細讀

pulseIn(pin, HIGH) 在這裡是完美的工具：它會阻塞直到接腳變 HIGH，啟動一個微秒計時器，並在接腳再度變 LOW 時回傳脈衝寬度（µs）。我們給它一個逾時，這樣當回聲缺席（物件太遠，或根本沒物件）時，迴圈才不會永遠卡住。

hcsr04.ino C++ / Arduino
const uint8_t TRIG = 9;
const uint8_t ECHO = 10;

void setup() {
  Serial.begin(115200);
  pinMode(TRIG, OUTPUT);
  pinMode(ECHO, INPUT);
  digitalWrite(TRIG, LOW);   // 在第一個脈衝前確保乾淨的 LOW
}

float measureCm() {
  // 1) 發出一個乾淨的 10 us 觸發脈衝。
  digitalWrite(TRIG, LOW);  delayMicroseconds(2);
  digitalWrite(TRIG, HIGH); delayMicroseconds(10);   // 規格書：>= 10 us
  digitalWrite(TRIG, LOW);

  // 2) 計時 ECHO 維持 HIGH 多久（來回飛行時間，單位 us）。
  //    30000 us 的逾時把量程上限設在約 5 m，並避免卡死。
  unsigned long t = pulseIn(ECHO, HIGH, 30000UL);
  if (t == 0) return -1.0;          // 逾時內沒有回聲

  // 3) 換算：d_cm = t_us / 58.3  (= 來回 0.0343 cm/us 的一半)。
  return t / 58.3f;
}

void loop() {
  float cm = measureCm();
  if (cm < 0) Serial.println(F("out of range"));
  else { Serial.print(cm, 1); Serial.println(F(" cm")); }
  delay(60);                        // 兩次 ping 至少間隔 60 ms：避免聽到
}                                   // 上一個啁啾的遲到回聲
⚠️ 陷阱——3.3 V 板子上的 5 V ECHO 線

經典的 HC-SR04 以 5 V 運作，並把 ECHO 驅動到 5 V。把它直接接進 3.3 V 的 ESP32 或 STM32 接腳，你會慢慢把它煮熟。請用電阻分壓把 ECHO 降下來（例如串聯 1 kΩ、對地 2 kΩ，得到 $5\cdot\tfrac{2}{3}=3.3$ V），或使用電平轉換器。TRIG 在模組上是輸入腳，所以它收到的 3.3 V HIGH 沒問題。

🔬 即時：回聲時間 → 距離

拖曳 ECHO 脈衝寬度，看著障礙物移動。讀數即時套用 d = t/58.3——正是你的韌體所計算的。

3單線時序：DHT11/DHT22 通訊協定

① 直覺 整段對話只用一條線

HC-SR04 交給你的是一個類比意味的脈衝寬度。許多數位感測器則是用一條資料線講一種位元敲擊式（bit-banged）的序列協定，其中每個 HIGH 脈衝的持續時間編碼一個位元。DHT11（溫度＋濕度，便宜，解析度低）與 DHT22（又稱 AM2302，解析度較細）就是經典範例，它們教你的這套模式，你也會在 WS2812 LED 以及許多其他元件上看到。

🔑 用文字說明 DHT 握手流程

1) MCU 把線拉 LOW 至少 18 ms（DHT11）以喚醒感測器，然後釋放它。2) 感測器以 80 µs LOW 接著 80 µs HIGH 回應。3) 它串流送出 40 個位元（16 位濕度、16 位溫度、8 位校驗和）。每個位元以 50 µs LOW 開始；接著 約 26 µs HIGH 表示 0、約 70 µs HIGH 表示 1。你靠計時那段 HIGH 來解碼。

sequenceDiagram participant MCU participant DHT MCU->>DHT: 拉 LOW >= 18 ms (起始) MCU->>DHT: 釋放 (上拉電阻把線帶到 HIGH) DHT->>MCU: 80 us LOW + 80 us HIGH (應答) loop 40 個位元 DHT->>MCU: 50 us LOW (位元起始) DHT->>MCU: HIGH ~26us = 0 / ~70us = 1 end Note over MCU: byte5 == (b0+b1+b2+b3) & 0xFF ? 有效

時序緊到（微秒等級）讓可靠讀取 DHT 意味著要在擷取期間關閉中斷，這正是為什麼你該用一個經過驗證的函式庫，而不是自己手刻。但你必須理解這個協定才能除錯：校驗和失敗通常代表某個中斷（一次 millis 計時、一個序列位元組）把那些 HIGH 視窗中的某一個拉長了，翻轉了一個位元。

② 數學 校驗和

那 40 個位元以五個位元組抵達。最後一個是校驗和：前四個位元組總和的低 8 位元。你用以下方式驗證

$$ \text{checksum} \;\overset{?}{=}\; (b_0 + b_1 + b_2 + b_3)\ \&\ \texttt{0xFF} $$
dht_decode.ino C++ / Arduino
// 對擷取到的 5 個位元組做概念性解碼（真正的驅動程式會先擷取
// 40 個邊緣的時序；這裡只示範驗證 + 換算）。
bool dhtDecode(const uint8_t b[5], float &rh, float &tC) {
  uint8_t sum = b[0] + b[1] + b[2] + b[3];   // 8 位元溢位捨棄是刻意的
  if (sum != b[4]) return false;             // 校驗和不符 -> 拒絕

  // DHT22：16 位元數值，0.1 單位解析度，溫度 MSB = 正負號。
  rh = ((b[0] << 8) | b[1]) * 0.1f;
  int16_t raw = ((b[2] & 0x7F) << 8) | b[3];
  tC = raw * 0.1f;
  if (b[2] & 0x80) tC = -tC;                  // 最高位元為 1 => 負值
  return true;
}
⚠️ 陷阱——取樣 DHT 太快

DHT11 約以 1 Hz 更新，DHT22 約 0.5 Hz。輪詢得更快，你只會拿到舊的或損毀的讀數。還有：資料線需要一個對 VCC 的 4.7–10 kΩ 上拉電阻，因為這個協定仰賴在雙方都不驅動時讓線浮在 HIGH。忘了上拉電阻，你就會得到全零或全一的垃圾。

4致動器與飛輪二極體：馴服電感反衝

① 直覺 為什麼線圈會反咬一口

繼電器、電磁閥和馬達裡都含有一個線圈——也就是電感。電感討厭電流的突然變化：它的整個本性就是要讓電流持續流動。所以你一把它切 OFF 的那一瞬間，磁場崩潰，線圈就會產生一個巨大的反向電壓突波（反電動勢，back-EMF），試圖維持那股電流。那道突波可達數百伏特，會在你的切換電晶體上跳電弧，或把你的繼電器接點熔焊在一起。

② 數學 電感的復仇

電感兩端的電壓正比於其電流變化的速度：

$$ v_L = L\,\frac{di}{dt} $$

當一顆電晶體把線圈切斷時，它試圖在數奈秒內把電流從運轉值降到零。當 $dt \to 0$，那個導數爆炸，$v_L$ 突波就大到嚇人。一顆飛輪二極體（又稱續流二極體，freewheeling diode）跨接在線圈兩端、在正常運作時反向偏壓，會在開關打開時給那股受困電流一個無害的迴圈去循環。電流此時便平緩地衰減為

$$ i(t) = I_0\,e^{-t/\tau}, \qquad \tau = \frac{L}{R} $$

而突波被箝制到大約只比電源高出一個二極體壓降（約 0.7 V），而不是數百伏特。每個線圈上一顆便宜的 1N4007。這沒得商量。

🔑 二極體方向——這個要對

二極體的陰極（有條紋的那端）接到 +V，陽極接到電晶體／線圈的交會點。所以正常運作時它是反向偏壓、形同隱形。只有當線圈把它的電壓甩到高於 +V 時它才導通，把電流引導成一個迴圈。裝反了，你就會透過一顆順向二極體把電源短路——瞬間冒煙。

🛠️ 交流負載需要緩衝器（snubber），不是二極體

飛輪二極體只對 DC 線圈有效。如果你的繼電器接點切換的是電感性的交流負載（市電馬達、變壓器），電弧發生在接點側，二極體幫不上忙——你要用一個 RC 緩衝器（例如 100 Ω + 100 nF）或在接點兩端跨一顆突波吸收器（varistor），來吸收電弧能量、阻止接點腐蝕。

5繼電器、光耦隔離與切換危險負載

① 直覺 用小訊號去指揮大訊號

繼電器是一個電控開關：一股小小的線圈電流把一個機械接點拉到閉合，讓一個完全獨立（而且可能是市電電壓）的電路導通。它的超能力是電氣隔離（galvanic isolation）——5 V 的控制側與 230 V 的負載側不共用任何銅箔，所以負載側的故障無法炸掉你的 MCU。標準的玩家用繼電器板還加了一個光耦合器（opto-coupler），連那顆線圈驅動電晶體都和你的微控制器做光學隔離：唯一跨越界線的東西是光。

③ 程式碼 低電位觸發的板子會咬新手

大多數玩家用繼電器板是低電位觸發（active-LOW）：當你把輸入腳驅動到 LOW 時繼電器才激磁，因為光耦合器的 LED 接在 VCC 與你的接腳之間。這也代表開機時、當接腳還在浮動的瞬間，繼電器可能會喀一聲 ON——請在把接腳設為 OUTPUT 之前先把它設成 HIGH（去激磁），以避免一次毛刺（glitch）。

relay.ino C++ / Arduino
const uint8_t RELAY = 7;   // 低電位觸發光耦板：LOW = ON, HIGH = OFF

void setup() {
  digitalWrite(RELAY, HIGH);   // 先去激磁，在 pinMode 之前...
  pinMode(RELAY, OUTPUT);      // ...這樣開機期間就不會喀一聲 ON
}

inline void relayOn()  { digitalWrite(RELAY, LOW);  }   // 注意：邏輯反相
inline void relayOff() { digitalWrite(RELAY, HIGH); }

void loop() {
  relayOn();  delay(2000);     // 負載通電 2 秒
  relayOff(); delay(2000);     // 然後斷電 2 秒
}
🏭 真實案例——隔離本身就是全部重點時

在一台工業 PLC 中，24 V 邏輯與它所指揮的 400 V 三相馬達，被刻意保持在分開的接地平面上，僅透過光耦合器與繼電器接點相連。理由既殘酷又具體：一次馬達故障可能會把一千伏特的暫態傾倒到它的配線上，而那道隔離屏障是唯一擋在這暫態與一台價值 50,000 美元控制器之間的東西。同樣的原則往下縮放到你那塊 3 美元的繼電器板：絕對不要讓市電與邏輯共用銅箔。

⚠️ 陷阱——市電不是教學玩具

230 V / 120 V 市電會要你的命。如果你對爬電距離（creepage）、外殼、保險絲與接地沒有十足把握，請改切換一個低壓直流負載（一台 12 V 風扇、一條 LED 燈條）來練習，把市電留給有資格的工作。韌體是一模一樣的；但一個閃失的後果可不一樣。

6直流馬達與 H 橋：方向＋速度

① 直覺 用四個開關讓馬達反轉

直流馬達在電流流過時往一個方向轉，電流反向時往另一個方向轉。要從單一電源讓電流反向，你需要一個 H 橋：四個開關像字母 H 一樣圍著馬達排列。閉合左上＋右下這一對，電流往一個方向流；閉合右上＋左下，電流往另一個方向流。像 L298N（較舊、BJT、約 1.7 V 壓降、每通道可達 2 A）或效率高得多的 DRV8833（MOSFET、低壓降、每通道可達約 1.5 A）這類驅動晶片，把兩個 H 橋包在一起，讓你能驅動兩顆馬達。

⚠️ 陷阱——直通（shoot-through）

絕對不要同時閉合 H 的同一側兩個開關（例如左上＋左下）。那會透過兩顆電晶體把電源直接短路到地——這叫「直通」——並在數微秒內摧毀整個橋。這個被禁止的輸入組合正是驅動晶片存在的理由：它們會為你強制插入死區時間（dead-time）並拒絕非法狀態。

② 數學 由 PWM 工作週期決定速度

H 橋設定方向；致能腳（enable）上的 PWM（第 4 章）設定速度。直流馬達的平均施加電壓是電源乘上工作週期，而一階近似下其無載轉速正比於該平均電壓：

$$ \bar{V}_{\text{motor}} = D \cdot V_{\text{supply}}, \qquad \omega \;\approx\; \frac{\bar{V}_{\text{motor}} - I\,R_a}{k_e} $$

其中 $D$ 是工作週期（0–1），$k_e$ 是馬達的反電動勢常數，$I R_a$ 是電樞電阻壓降。結論：工作週期就是你的油門。在 12 V 電源上、$D = 0.5$ 時，馬達看到的平均電壓約 6 V，運轉於接近其無載轉速的一半（在負載下會比較軟，因為有 $IR_a$ 這一項）。H 橋電晶體也必須能配合 PWM，這正是為什麼我們切換的是致能線，而不是方向線。

③ 程式碼 用 DRV8833 風格驅動器做方向＋速度

這裡我們用 L298N 的慣例：兩個方向輸入（IN1、IN2）設定極性，而一個支援 PWM 的 ENA 腳透過 analogWrite 設定速度。程式碼下方的真值表就是整個心智模型。

hbridge.ino C++ / Arduino
const uint8_t IN1 = 4, IN2 = 5;   // 方向控制
const uint8_t ENA = 6;            // PWM 速度 (在 Uno 上腳 6 = Timer0 OC0A)

void setup() {
  pinMode(IN1, OUTPUT); pinMode(IN2, OUTPUT); pinMode(ENA, OUTPUT);
}

// speed: 0..255 PWM 工作週期；dir: +1 前進, -1 後退, 0 = 滑行/煞車
void drive(int speed, int dir) {
  if (dir > 0)      { digitalWrite(IN1, HIGH); digitalWrite(IN2, LOW);  }
  else if (dir < 0) { digitalWrite(IN1, LOW);  digitalWrite(IN2, HIGH); }
  else              { digitalWrite(IN1, LOW);  digitalWrite(IN2, LOW);  } // 滑行
  analogWrite(ENA, constrain(speed, 0, 255));   // 工作週期 = speed/255
}

void loop() {
  drive(200, +1); delay(1500);   // 前進，約 78% 工作週期
  drive(0,    0); delay(400);    // 滑行
  drive(120, -1); delay(1500);   // 後退，約 47% 工作週期
  drive(0,    0); delay(400);
}

LOWHIGH← 後退電流 B→A
LOWLOW滑行兩個低側皆關 → 自由旋轉
HIGHHIGH煞車兩端短路 → 快速停止
🔑 伺服機 vs. 直流馬達 vs. 步進馬達

玩家用伺服機（PWM 那章）：你用每 20 ms 一次的 1–2 ms 脈衝去命令一個角度；它內建自己的 H 橋與回授。直流馬達：你命令方向＋速度；H 橋由你提供。步進馬達：透過一個依序切換的多線圈驅動器（A4988/DRV8825）做離散步進。不同的任務、不同的驅動器——但它們每一個都需要飛輪保護，因為它們每一個都是線圈。

7電源紀律：分開供電、共用接地

① 直覺 為什麼馬達一啟動你的 MCU 就重置

最常見的新手翻車：機器人往前開、Arduino 就重開機。原因：馬達的湧入電流（在堵轉／啟動時，它可能瞬間拉到運轉電流的 5–10 倍）把共用的 5 V 軌壓垮到掉電（brown-out）門檻以下，於是 MCU 重置。解法是電源紀律。

🔑 混合訊號供電的三條規則

1. 把馬達電源分開。馬達／繼電器要從它們自己的電池或穩壓器供電，絕不要走 MCU 的 5 V 腳。
2. 共用接地。所有電源的接地必須綁在一起——否則控制訊號就沒有共同的參考點，H 橋讀到的是垃圾。這是新手會跳過、然後說不出為什麼一切都不動的那條規則。
3. 處處去耦。每顆晶片的 VCC 旁靠近處放一顆 100 nF 陶瓷電容，再在馬達電源上跨一顆 100–470 µF 的大電解電容，用來吸收切換暫態。

flowchart TB BAT["電池 / 馬達電源"] --> REG["5V 穩壓器"] REG --> MCU["MCU VCC"] BAT --> DRV["馬達驅動器 VM"] MCU -. "訊號" .-> DRV GND([共用接地]) --- MCU GND --- DRV GND --- BAT
② 數學 編列電流預算

在選擇電源之前，先把最壞情況的電流加總起來。以一台雙馬達機器人為例：兩顆馬達堵轉、一個聲納，加上 MCU：

$$ I_{\text{total}} = \sum_k I_k = 2\,(0.8\,\text{A}) + 0.015\,\text{A} + 0.05\,\text{A} \approx 1.67\,\text{A} $$

挑一個額定值舒適地高於它的電源——比方說 2.5 A——這樣暫態才不會把電軌拉垮。然後把你的導線與驅動器的電流額定值依同一份預算來選。L298N 上限約每通道 2 A，而且每通道會以熱的形式浪費約 1.7 V；DRV8833 則能把你電池裡更多的能量留在馬達上。電流編列就是「能跑一整個下午的機器人」與「每次轉彎就重置的機器人」之間的差別。

8真實案例：一台避障機器人

④ 真實案例 這就是整章濃縮成一個迴圈

以上所有東西組合成一台小小的自主機器人：一個 HC-SR04 往前看，韌體做決策，一個 H 橋操控兩顆直流馬達。感測 → 決策 → 動作——就是 §1 那條訊號鏈，化為現實。

avoider.ino C++ / Arduino
// 接腳：聲納在 9/10，左馬達 IN1/IN2 + ENL，右馬達 IN3/IN4 + ENR。
const uint8_t TRIG=9, ECHO=10;
const uint8_t IN1=4, IN2=5, ENL=6, IN3=7, IN4=8, ENR=11;
const float   STOP_CM = 25.0f;     // 障礙物比這更近就轉開

float ping() {
  digitalWrite(TRIG, LOW);  delayMicroseconds(2);
  digitalWrite(TRIG, HIGH); delayMicroseconds(10);
  digitalWrite(TRIG, LOW);
  unsigned long t = pulseIn(ECHO, HIGH, 30000UL);
  return t ? t / 58.3f : 999.0f;   // 沒有回聲 => 視為「很遠」
}

void wheels(int l, int lDir, int r, int rDir) {
  digitalWrite(IN1, lDir > 0); digitalWrite(IN2, lDir < 0); analogWrite(ENL, l);
  digitalWrite(IN3, rDir > 0); digitalWrite(IN4, rDir < 0); analogWrite(ENR, r);
}

void setup() {
  pinMode(TRIG, OUTPUT); pinMode(ECHO, INPUT);
  uint8_t outs[] = {IN1,IN2,ENL,IN3,IN4,ENR};
  for (uint8_t p : outs) pinMode(p, OUTPUT);
}

void loop() {
  if (ping() < STOP_CM) {           // 前方有障礙：原地往右旋轉
    wheels(180, +1, 180, -1);
    delay(350);
  } else {                          // 路徑淨空：以約 70% 工作週期前進
    wheels(180, +1, 180, +1);
  }
  delay(60);                        // 遵守 >=60 ms 的聲納間隔
}
🏭 真實案例——這套模式真的出貨在實際產品裡

掃地機器人正是這個迴圈放大版：懸崖感測器與保險桿餵給一個決策層，由它指揮一個雙 H 橋來驅動行走輪，並用一個無刷驅動器來驅動吸力馬達——每一個電感性負載都有自己的續流保護，全都掛在一棵以星形接地、由一顆電池透過分開的穩壓器供電的電源樹上。那台 20 美元的桌面機器人與那台 400 美元的掃地機，跑的是你剛讀過的同一副骨架。掌握這副骨架，你兩者都能造。

9⑤ 換你了

⑤ 練習 你來寫、你來決定

三道檢測。點一個答案、輸入一個數字，或揭開一道程式挑戰。你的進度會儲存在這台裝置上，並顯示在首頁。

Q1 觀念檢測
你把一顆直流馬達透過 H 橋接到一顆外部 9 V 電池，Arduino 用它自己的 5 V 電源。馬達隨機抽搐，橋讀到的是垃圾指令。你加了一顆飛輪二極體，它還是表現失常。最可能的剩餘錯誤是什麼？
  • 飛輪二極體太小——換一顆大一點的
  • 兩個電源的接地沒有綁在一起（沒有共用接地）
  • 馬達需要更高的 PWM 頻率
  • Arduino 接腳無法為馬達提供足夠電流
使用分開的電源時你必須把它們的接地接在一起。邏輯訊號是相對於接地的電壓；沒有共同參考點，驅動器就分不出 HIGH 與 LOW，讀到的都是垃圾。飛輪二極體防的是反電動勢（那是另一個問題），而 Arduino 接腳只驅動驅動器的邏輯輸入，不是馬達本身——所以缺的那一塊是共用接地。
Q2 算算看
你的 HC-SR04 的 ECHO 脈衝量到 1740 µs 寬。障礙物有多遠，以公分表示？（使用規則 $d_{\text{cm}} = t_{\mu s}/58$。）
cm
$d = 1740 / 58 = 30\ \text{cm}$。用 SI 交叉檢查：$d = v\,t/2 = 0.0343\,(\text{cm}/\mu s) \times 1740\,\mu s / 2 = 29.9\ \text{cm}$。「除以 58」這個規則其實就是 $2/0.0343$ 四捨五入——快到足以每個迴圈都跑。
Q3 程式挑戰——用 pulseIn 算距離
為一個 TRIG = 3、ECHO = 2 的 HC-SR04 寫一個函式 float distanceCm()。它必須發出一個正確的 10 µs 觸發脈衝，用逾時計時回聲，並回傳公分（若沒有回聲抵達則回傳 -1）。然後每秒印出一次讀數。
提示
流程是：把 TRIG 短暫驅動 LOW，HIGH 維持 10 µs（delayMicroseconds(10)），再回到 LOW。接著 pulseIn(ECHO, HIGH, 30000) 會給出寬度（µs）；除以 58.3。pulseIn 回傳 0 代表逾時到期——把它轉換成你的 -1 哨兵值。
顯示一種解答
distance.ino C++ / Arduino
const uint8_t TRIG = 3;
const uint8_t ECHO = 2;

void setup() {
  Serial.begin(115200);
  pinMode(TRIG, OUTPUT);
  pinMode(ECHO, INPUT);
  digitalWrite(TRIG, LOW);
}

float distanceCm() {
  digitalWrite(TRIG, LOW);  delayMicroseconds(2);
  digitalWrite(TRIG, HIGH); delayMicroseconds(10);   // 10 us 觸發
  digitalWrite(TRIG, LOW);

  unsigned long t = pulseIn(ECHO, HIGH, 30000UL);    // us, 約 5 m 上限
  if (t == 0) return -1.0f;                          // 沒有回聲
  return t / 58.3f;                                  // d_cm = t_us / 58.3
}

void loop() {
  float cm = distanceCm();
  if (cm < 0) Serial.println(F("no echo"));
  else { Serial.print(cm, 1); Serial.println(F(" cm")); }
  delay(1000);
}

逾時是那個生產級的細節：少了它，缺席的回聲會讓 pulseIn 阻塞長達一秒，你的控制迴圈就停擺了。封頂在 30 ms 把最壞情況限制在約 5 m 的量程，並讓迴圈保持反應靈敏。

🎓 本章回顧

你現在能夠推理從感測器到致動器的一整條訊號鏈：對 HC-SR04 的回聲計時並用 $d_{\text{cm}} = t_{\mu s}/58.3$ 換算成距離；解碼一個 DHT 單線訊框並驗證它的校驗和；用一顆方向正確的飛輪二極體保護每一個電感性負載（$v_L = L\,di/dt$ 就是原因），並在交流接點上改用 RC 緩衝器；把危險負載放在光耦隔離的繼電器後方來切換；用 H 橋驅動直流馬達的方向、用 PWM 工作週期控制它的速度，同時避免直通；並用分開的電源、一個共用接地、去耦電容，以及一份誠實的電流預算，讓整套系統存活下來。這就是一台小型自主機器人所需的一切——感測、決策、動作，並在自己的致動器手下倖存。
