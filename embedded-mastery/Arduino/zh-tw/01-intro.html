基礎篇：微控制器、AVR 與你的第一支 Sketch

在你寫下任何一行程式之前，必須先對自己究竟在對什麼寫程式建立正確的心智模型。我們會從矽晶片由下而上建構這個模型——接著讓一顆 LED 閃爍，並徹底理解它為何能運作的每一個奈秒。

1微控制器究竟是什麼？

① 直覺 先建立畫面

你的筆電是一台電腦：一顆 CPU，加上負責記憶體、儲存、顯示與 I/O 的數顆獨立晶片，全部佈線在一塊大電路板上。微控制器（MCU）則把這整個概念縮進一顆晶片裡——CPU、記憶體與周邊裝置融合成一片指甲大小的矽晶。它只要幾塊美金、耗電以毫瓦計，並在數微秒內開機，完全看不到作業系統的蹤影。

💡 一句話模型

微控制器是一台微小、自給自足、永遠只執行一支程式的電腦，透過它的接腳直接與物理世界對話。

你桌上的「Arduino」是一塊圍繞著 MCU 打造的方便開發板。在經典的 Arduino Uno 上，那顆 MCU 是 Microchip 的 ATmega328P，一顆 8 位元的 AVR 晶片。開發板額外加上 USB 接頭、穩壓器、時脈石英晶體與排針——這些都是方便用的配件。真正的大腦是 328P。

🔑 關鍵區別

微處理器（如 PC 裡那顆）需要外部晶片才能做任何事。微控制器本身就已內含記憶體與周邊裝置——它一通電的瞬間就「準備好掌控物理世界」。

2ATmega328P 的解剖學

① 直覺 裡面有什麼

有三件事主導你怎麼寫嵌入式程式：CPU 跑多快、你能用多少記憶體，以及哪種記憶體存放什麼。328P 刻意做得很小——而這份稀缺性塑造了你日後的每一個決策。

資源ATmega328P（Uno）你的筆電比例
時脈速度16 MHz約 3 GHz約慢 190 倍
程式記憶體（Flash）32 KB約 512 GB SSD約 1600 萬倍
RAM（SRAM）2 KB約 16 GB約 800 萬倍
CPU 位元寬度8 位元64 位元—
🔑 三種記憶體，三種職責

Flash（32 KB）——非揮發性；存放你編譯後的程式。斷電後依然保留。
SRAM（2 KB）——揮發性；執行期間存放變數。斷電即消失。
EEPROM（1 KB）——非揮發性；存放你選擇要永久保留的設定（例如一個校正值）。

AVR 採用 哈佛架構（Harvard architecture）：程式（Flash）與資料（SRAM）位於各自獨立的匯流排與位址空間。這就是為什麼字串常數 "hello" 暗地裡住在 Flash 裡，卻會在開機時被複製到你寶貴的 SRAM——這是「我怎麼會把 RAM 用完了？」這類意外的經典來源，我們會在第 8 章拆解它。

② 數學 編列稀缺記憶體的預算

記憶體計算在嵌入式工作中無所不在。假設你把溫度取樣存成 16 位元整數（int16_t＝每筆 2 個位元組）。在忽略額外開銷的情況下，SRAM 能容納幾筆？

$$ N_{\max} = \left\lfloor \frac{\text{SRAM}}{\text{bytes per sample}} \right\rfloor = \left\lfloor \frac{2048 \text{ B}}{2 \text{ B}} \right\rfloor = 1024 \text{ samples} $$

實際上，堆疊（stack）、全域變數與 Arduino 核心函式庫早已占去數百個位元組——所以要預留餘裕。這種信封背面快速估算的習慣，正是「能運作好幾年的韌體」與「凌晨三點神祕自我損毀的韌體」之間的差別。

🛠️ IDE 替你藏起來的事

你的 .ino 檔其實並不完全是 C++。IDE 會把你各分頁串接起來、自動產生函式原型、用 Arduino 核心包住一切，並插入一個隱藏的 main()。所謂的「魔法」不過就是預處理——你完全可以用 arduino-cli compile 在命令列重現這一切。

🏭 真實案例——Bootloader

一顆全新的晶片，在沒有燒錄器的情況下，怎麼能透過單純的 USB 接受程式碼？有一支稱為 bootloader（開機載入程式） 的小程式，住在 Flash 受保護的一個角落。重置時它會聆聽約 1 秒；如果 IDE 正在傳輸，它就把進來的位元組寫入 Flash，然後跳到你的程式。這 1 秒的聆聽，正是 Uno 在通電後要愣一下才啟動的原因。

4Sketch 的解剖學：setup() 與 loop()

① 直覺 先跑一次，然後永遠跑下去

每一支 Arduino sketch 都恰好有兩個必備函式。setup() 在開機時只跑一次——在這裡設定你的接腳。接著 loop() 會一遍又一遍、永不停止地，以 CPU 能達到的最快速度執行。這裡沒有 return、沒有「程式結束」——嵌入式軟體永遠不會停。

🛠️ 隱藏的 main()

在幕後，Arduino 核心其實確實有一個標準的 C++ main()。它大致是：int main(){ init(); setup(); for(;;){ loop(); } }。所以 loop() 實際上是在一個無窮的 for(;;) 裡被呼叫。你寫的每一行，永遠都在一個永恆的迴圈之中。

5你的第一支程式：Blink，逐行完整註解

③ 程式碼 讀懂每一行
blink.ino C++ / Arduino
// Uno 有一顆內建 LED，接在數位接腳 13。
// LED_BUILTIN 是一個具名常數，在 Uno 上等於 13。

void setup() {
  // 只跑一次。把接腳 13 設定為 OUTPUT，這樣 MCU
  // 才能在它上面「驅動」電壓（而不是「聆聽」）。
  pinMode(LED_BUILTIN, OUTPUT);
}

void loop() {
  // 永遠執行。
  digitalWrite(LED_BUILTIN, HIGH);  // 接腳給 5 V → LED 亮
  delay(500);                       // 什麼都不做，停 500 ms
  digitalWrite(LED_BUILTIN, LOW);   // 接腳給 0 V → LED 滅
  delay(500);                       // 再等 500 ms
}                                   // loop() 返回 → 核心再次呼叫它
💡 每個呼叫實際上做了什麼

pinMode(pin, OUTPUT) 會翻轉硬體暫存器裡的一個位元，讓接腳變成一個微小、可控的電壓源。digitalWrite(pin, HIGH) 把接腳設為約 5 V；LOW 則設為 0 V。delay(ms) 讓 CPU 在一個計數迴圈裡空轉，做那麼多毫秒的無用功。（那種「什麼都不做」是我們會在第 5 章懺悔的罪過。）

⚠️ 陷阱——忘了 pinMode

如果你略過 pinMode(.., OUTPUT)，接腳會維持在輸入狀態，而 digitalWrite(.., HIGH) 會悄悄啟用內部的提升電阻（pull-up resistor），而不是驅動該接腳。你的 LED 會微亮或全暗，而且不會跳出任何錯誤。永遠先設定好方向。

6時序數學：時脈、週期，以及 16 MHz 的意義

② 數學 時間從何而來

Uno 的石英晶體以 $f_{\text{clk}} = 16\,\text{MHz} = 16{\times}10^6$ 個週期每秒的頻率振盪。每一條指令需要一個或多個時脈週期（clock cycle）。單一週期的時間長度就是它的倒數：

$$ T_{\text{cycle}} = \frac{1}{f_{\text{clk}}} = \frac{1}{16\times10^6\,\text{Hz}} = 62.5\,\text{ns} $$

所以在你的 LED 亮著的那半秒裡，CPU 大約執行了

$$ N = f_{\text{clk}} \cdot t = 16\times10^6 \,\tfrac{\text{cyc}}{\text{s}} \times 0.5\,\text{s} = 8\times10^6 \text{ cycles} $$

——八百萬個週期，幾乎全部都耗在 delay() 裡，徹底地什麼也沒做。現在反過來看：我們這個方波的閃爍頻率與占空比（duty cycle）為

$$ T = t_{\text{on}} + t_{\text{off}}, \qquad f = \frac{1}{T}, \qquad D = \frac{t_{\text{on}}}{T}\times100\% $$

當 $t_{\text{on}} = t_{\text{off}} = 500\,\text{ms}$ 時：週期 $T = 1\,\text{s}$、頻率 $f = 1\,\text{Hz}$、占空比 $D = 50\%$。記牢這條公式——它是第 4 章 PWM 的種子，屆時我們會閃得太快，讓眼睛看到的是亮度而非閃動。

7真實案例：不起眼的狀態 LED

④ 真實案例 這會延伸到實際產品

「Blink」感覺像玩具，但閃爍的狀態 LED 在出貨硬體裡無所不在——路由器、3D 印表機、醫療裝置、人造衛星。它是成本最低的人類可讀診斷工具：一台沒有螢幕的裝置，仍然能用一顆 LED、零額外成本告訴你「開機中」（快閃）、「執行中」（緩慢脈動）或「故障」（SOS 模式）。

🏭 真實案例——把心跳當看門狗

成熟的韌體常會在主迴圈內部切換一顆「心跳（heartbeat）」LED。如果那顆 LED 停止閃爍，工程師立刻就知道迴圈卡住了——完全不需要除錯器。這個閃爍模式本身就是存活訊號。你剛剛學會了一個真正的量產技巧。

Q1 觀念檢核
你在 loop() 內定義了 int sensorValue = 0;。當程式執行時，這個變數實際上住在哪裡？
  • Flash 記憶體
  • SRAM
  • EEPROM
  • Bootloader
執行期間的變數住在 SRAM（揮發性，Uno 上有 2 KB）。Flash 存放程式；EEPROM 存放你明確選擇要永久保留的設定。這正是哈佛架構分割在實際運作。
Q2 算算看
另一塊 AVR 開發板以 16 MHz 運行。一個時脈週期有多長，以奈秒為單位？（回想 $T_{\text{cycle}} = 1/f_{\text{clk}}$，且 $1\,\text{s}=10^9\,\text{ns}$。）
ns
$T = \dfrac{1}{16\times10^6} = 62.5\times10^{-9}\,\text{s} = 62.5\,\text{ns}$。在 8 MHz 時會是 125 ns；在 1 MHz 時則是 1000 ns。週期時間就只是時脈頻率的倒數。
Q3 程式挑戰——摩斯電碼 SOS
修改 Blink，讓 LED 打出 SOS：三短閃（·）、三長閃（–）、三短閃（·），然後一段長暫停。短＝亮 200 ms，長＝亮 600 ms，閃與閃之間的間隔＝滅 200 ms。想想看，要怎麼避免把 digitalWrite 複製貼上九次。
提示
寫一個輔助函式 flash(int onMs)，讓 LED 亮 onMs、再滅 200 ms。然後用 200 呼叫它三次、用 600 三次、用 200 三次，並在 loop() 結尾加上一個長的 delay(700)。
顯示一種解法
sos.ino C++ / Arduino
const int LED = LED_BUILTIN;

void setup() { pinMode(LED, OUTPUT); }

// 一次閃爍：亮 onMs，接著一段 200 ms 的間隔
void flash(int onMs) {
  digitalWrite(LED, HIGH); delay(onMs);
  digitalWrite(LED, LOW);  delay(200);
}

void loop() {
  for (int i = 0; i < 3; i++) flash(200);   // S : · · ·
  for (int i = 0; i < 3; i++) flash(600);   // O : – – –
  for (int i = 0; i < 3; i++) flash(200);   // S : · · ·
  delay(700);                                // 字間間隔
}

重點在 flash() 這個輔助函式：把重複的部分抽取成一個函式。這同時也預示了為什麼 delay() 有局限——真實產品需要在閃爍的同時讀取按鈕，而會阻塞的 delay() 不容許這樣做。第 5 章正是要修正這一點。
